# TKT-024: `docket check --fix` skips other rules after autofix

## Description
When `docket check --fix` removes a resolved blocker, the checker immediately `continue`s
to the next ticket instead of re-running the rest of the rule set on the updated ticket.
That can hide unrelated findings such as stale state or schema errors.

## Acceptance Criteria
- [x] `docket check --fix` still reports non-R006 findings on the same ticket after removing resolved blockers : Implemented and verified by tests in this change set.
