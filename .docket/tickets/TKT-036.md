# TKT-036: Add semantic provider registry

## Description
Why: provider selection should be centralized instead of spread across future commands and indexer code.

Scope:
- Add a registry or factory that resolves a provider by config name.
- Return deterministic errors for unknown provider names.

Files to touch:
- Semantic provider package and config-to-provider wiring.

Non-goals:
- No real `uv` subprocess behavior beyond what is needed to compile.

## Acceptance Criteria
- [ ] Provider names resolve through a single registry instead of `switch` statements spread through the codebase.
- [ ] Unknown provider names return a structured error that can be surfaced by future CLI commands.
