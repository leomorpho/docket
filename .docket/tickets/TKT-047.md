# TKT-047: Add provider timeout handling

## Description
Why: first-run model downloads and local Python startup can hang or take much longer than normal indexing work.

Scope:
- Add timeout control around provider subprocess execution.
- Distinguish timeout failures from normal provider errors.

Files to touch:
- `uv` provider, subprocess helpers, and provider tests.

Non-goals:
- No retries or background downloads.

## Acceptance Criteria
- [x] Stalled `uv` or Python embedding work fails within a bounded timeout instead of hanging the CLI. : UV provider execution is now wrapped in bounded timeouts for status and embed calls.
- [x] Timeout failures are distinguishable from normal provider execution errors. : Provider tests verify timeout failures are surfaced as a distinct provider error kind.
