# TKT-047: Add provider timeout handling

## Description
Why: first-run model downloads and local Python startup can hang or take much longer than normal indexing work.

Scope:
- Add timeout control around provider subprocess execution.
- Distinguish timeout failures from normal provider errors.

Files to touch:
- `uv` provider, subprocess helpers, and provider tests.

Non-goals:
- No retries or background downloads.

## Acceptance Criteria
- [ ] Stalled `uv` or Python embedding work fails within a bounded timeout instead of hanging the CLI.
- [ ] Timeout failures are distinguishable from normal provider execution errors.
