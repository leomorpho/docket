# TKT-048: Add provider stderr and error parsing

## Description
Why: semantic CLI commands will be hard to debug if local provider failures are flattened into one generic error.

Scope:
- Parse stderr and bridge output into structured provider errors.
- Distinguish non-zero exit, invalid JSON, and bridge-declared errors.

Files to touch:
- `uv` provider, bridge contract types, and provider tests.

Non-goals:
- No ranking or index persistence changes.

## Acceptance Criteria
- [ ] Bridge failures surface useful structured errors instead of raw opaque subprocess failures.
- [ ] Malformed JSON, Python exceptions, and non-zero exits are distinguishable in provider results.
