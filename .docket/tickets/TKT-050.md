# TKT-050: Define semantic chunk ID format

## Description
Why: incremental upserts and deletes require stable chunk identifiers.

Scope:
- Define a deterministic chunk ID format.
- Encode ticket ID, field identity, and repeated-item position where needed.

Files to touch:
- Semantic chunking package and unit tests.

Non-goals:
- No hash algorithm or storage schema.

## Acceptance Criteria
- [ ] Each semantic chunk gets a deterministic ID that remains stable across rebuilds when ticket content is unchanged.
- [ ] The ID format can distinguish ticket, field, and repeated item position without relying on database row IDs.
