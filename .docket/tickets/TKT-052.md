# TKT-052: Implement semantic ticket chunker

## Description
Why: the indexer and related-ticket command need one shared chunk generation path.

Scope:
- Convert a ticket into ordered semantic chunks with IDs, types, and hashes.
- Handle missing optional fields without placeholder chunks.

Files to touch:
- Semantic chunking package and any glue code that reads ticket objects.

Non-goals:
- No persistence or provider calls.

## Acceptance Criteria
- [ ] A ticket can be converted into an ordered list of deterministic semantic chunks with IDs, types, and hashes.
- [ ] Chunking behavior handles missing optional fields without panics or placeholder chunks.
