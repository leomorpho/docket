# TKT-064: Add incremental semantic diff detector

## Description
Why: incremental rebuilds are the main performance benefit of local semantic indexing.

Scope:
- Compare freshly chunked tickets to persisted semantic metadata.
- Classify each chunk as add, change, delete, or unchanged.

Files to touch:
- Semantic indexer package and metadata-store tests.

Non-goals:
- No provider calls or vector writes.

## Acceptance Criteria
- [ ] Current ticket chunks can be compared to stored semantic metadata and classified as add, change, delete, or unchanged.
- [ ] Diff output is deterministic and ready for the future incremental indexer to consume directly.
