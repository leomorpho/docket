# TKT-072: Add index stats to semantic status

## Description
Why: users need to know whether a local semantic index exists and whether it is fresh enough to trust.

Scope:
- Print chunk count, ticket count, index freshness, model identity, and last update time in `semantic status`.
- Source the data from the semantic metadata store and freshness calculation.

Files to touch:
- Semantic status command and metadata/freshness plumbing.

Non-goals:
- No rebuild execution.

## Acceptance Criteria
- [ ] `semantic status` shows chunk count, ticket count, freshness, model identity, and last update time when available.
- [ ] Missing or stale index states are surfaced explicitly instead of appearing as blank successful output.
