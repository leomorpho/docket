# TKT-106: Make handoff section structure config-driven with template support

## Description
## Goal
Replace hardcoded required handoff subsections in `internal/store/local/validate.go` with a config-driven template, allowing teams to customize what sections a handoff must contain.

## Current hardcoded code (internal/store/local/validate.go lines 194-209)
```go
requiredSections := []string{
  "**Current state:**",
  "**Decisions made:**",
  "**Files touched:**",
  "**Remaining work:**",
  "**AC status:**",
}
// validates that handoff field contains all these substrings
```

Also hardcoded in `cmd/session_compress.go`: the LLM prompt that generates handoffs references specific section names.

## Config schema addition (add to config.json)
```json
{
  "handoff_sections": [
    "Current state",
    "Decisions made",
    "Files touched",
    "Remaining work",
    "AC status"
  ]
}
```

The validator should check that the handoff field contains each section name (case-insensitive, wrapped in ** or ## or similar markdown). Keep the matching flexible — check for presence of the string, not exact markdown format.

## What to do
1. Add `HandoffSections []string` to the `Config` struct in `internal/ticket/config.go`.
2. Set default in `applyDefaults()`: the 5 sections above.
3. In `internal/store/local/validate.go`: replace hardcoded `requiredSections` with `cfg.HandoffSections`. Load config at the start of validation (it may already be available — check how validate.go is called).
4. In `cmd/session_compress.go`: build the LLM prompt section list from `cfg.HandoffSections` instead of hardcoding section names in the prompt string.
5. The matching logic: for each required section, check `strings.Contains(strings.ToLower(handoff), strings.ToLower(section))`.

## Files to change
- `internal/ticket/config.go` — add HandoffSections field + default
- `internal/store/local/validate.go` — use cfg.HandoffSections
- `cmd/session_compress.go` — build prompt from cfg.HandoffSections

## Acceptance criteria
- [ ] handoff_sections in config.json controls which sections are required in ticket handoffs
- [ ] Removing a section from config causes validate to stop requiring it
- [ ] Adding a custom section causes validate to require it
- [ ] Default config (no handoff_sections key) applies the 5 standard sections
- [ ] session compress LLM prompt uses sections from config
