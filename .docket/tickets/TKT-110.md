# TKT-110: Board view with config-driven columns and ticket cards

## Description
Build the kanban board view on top of the scaffold from TKT-109.

## Acceptance Criteria
- [x] Columns render in config-defined order (column field), only open=true states shown : BoardView builds columns from config state entries sorted by column and only open=true states
- [x] Each column header shows state label and ticket count : Column headers render state label and per-column ticket counts
- [x] TicketCard shows title (2-line clamp), ID badge, priority badge, labels : TicketCard renders title clamp, ID badge, priority badge, and labels
- [x] Board is horizontally scrollable; each column scrolls independently vertically : Board container has overflow-x auto and each column body has independent overflow-y auto
- [x] Clicking a card emits a select event (detail panel integration comes in TKT-111) : TicketCard dispatches select event; BoardView forwards select event on click for panel integration
