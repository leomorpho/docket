# TKT-118: Smarter direct-edit validation: accept valid, reject+educate invalid

## Description
When docket sync/validate detects direct-edited ticket files, validate each against the full schema. If valid: accept the changes but print a warning showing the equivalent CLI command (e.g. 'docket update TKT-NNN --state done'). If invalid: revert the changes, clearly explain the schema violation, and show the CLI command that would achieve what they were trying to do. Never silently drop changes or silently accept schema violations.
