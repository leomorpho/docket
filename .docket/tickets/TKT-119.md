## Description
The agent_instructions field in help-json output is read by LLMs before they use docket. Expand it from a single line into a structured guide covering: (1) keep tickets atomic - one deliverable, completable in a single focused session; (2) description must explain context and constraints, not just the task name; (3) every ticket needs at least 2 acceptance criteria that are specific and testable; (4) if a task would touch more than 3 files or take multiple logical steps, split it into children under a parent epic; (5) cheap/small LLMs should be able to understand and execute the ticket without asking clarifying questions.
