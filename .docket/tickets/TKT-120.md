# TKT-120: docket create quality validation: enforce description and AC minimums

## Description
Add validation to docket create: (1) require --desc or reject with a helpful message; (2) warn if description is under 50 characters - it is probably just restating the title; (3) after ticket creation, prompt or remind the user to add acceptance criteria via docket ac add. Optionally support --ac flag on create to add first AC inline. The goal is to make it hard to create low-quality tickets accidentally.
