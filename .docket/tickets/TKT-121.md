# TKT-121: docket validate quality checks: AC count, desc length, reasoning trail

## Description
Extend docket validate beyond schema checks to ticket quality: warn on tickets with 0 AC items, descriptions under a word threshold (e.g. 20 words), in-progress tickets with no comments (no reasoning trail). These are soft warnings, not errors - they do not fail the exit code unless a --strict flag is passed. Helps catch tickets that are too vague for an LLM to execute without guesswork.
