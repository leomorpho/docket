# TKT-122: Version update notifications: detect and surface new docket releases

## Description
Embed a version string at build time via ldflags. On any docket command, check ~/.cache/docket/version_check (a file with timestamp + latest version). If the file is older than 24h, fetch the GitHub releases API in the background and update the cache. If the cached latest version is newer than the embedded version, print a one-line notice at the end of output with the upgrade command. The background check must never add latency to the main command.
