# TKT-131: docket install — git hook infrastructure

## Description
Implement docket install command that wires docket into a git repo's pre-commit hook and injects a managed instructions block into CLAUDE.md. This is the foundation all other enforcement features depend on.

The command must:
- Write .git/hooks/pre-commit (create or append, mark executable)
- Write a versioned manifest to .docket/install.json recording docket version, install date, and managed artifact paths
- Inject a delimited block into CLAUDE.md (or create it) between <!-- docket:start --> and <!-- docket:end --> markers
- The pre-commit hook must detect if the commit message references a ticket ID (Ticket: TKT-NNN) and warn (non-blocking initially) if missing

Related to upcoming tickets for: docket upgrade (versioned artifact management), human interactive AC prompt in pre-commit hook, and scope locking.

Refined hook enforcement logic: hook should NOT block commits with no ticket reference (backward compatible). It SHOULD block if the current commit message references a ticket that is already in 'done' state. Only the current commit message is inspected — no git history traversal.

### 2026-03-12T02:39:53Z — agent:claude-sonnet-4-6
AC 4 clarification — 'warns if no Ticket: TKT-NNN found' should be read as a non-blocking warning only (stdout message, exit 0). The blocking enforcement is captured in AC 6: block only when a referenced ticket is already in 'done' state. These are two distinct behaviors: missing reference = warn and pass; stale/done reference = hard block.
