# TKT-132: docket upgrade — versioned artifact management

## Description
Implement docket upgrade command that updates installed artifacts (hooks, CLAUDE.md block) without overwriting user-owned config. Also implement a migration runner for breaking changes between docket versions.

Rules:
- .git/hooks/pre-commit — always overwrite (docket-owned)
- CLAUDE.md docket block — replace only between <!-- docket:start --> / <!-- docket:end --> markers
- .docket/config.yaml — never overwrite; only add missing keys with comments
- User content outside docket markers — never touch

Migrations live in src/migrations/ keyed by semver, run in order on upgrade. Each migration is a transform function over .docket/ files.

Depends on TKT-131 (docket install — git hook infrastructure).
