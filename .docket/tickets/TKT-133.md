# TKT-133: Runnable verification commands on AC items

## Description
Extend AC items to support an optional run: field containing a shell command. When docket ac check TKT-NNN runs, it executes any run: commands, captures exit codes and stdout, and uses them as pass/fail evidence.

Usage: docket ac add TKT-NNN --desc "Tests pass" --run "pnpm test --coverage"

The check command should:
- Run each run: command in the repo root
- Capture exit code and last 20 lines of stdout as evidence
- Auto-populate evidence on pass; mark failed with captured output
- Support --dry-run to show what would be executed without running

This feature is foundational for TKT-D (smart stack defaults), TKT-E (human interactive AC prompt), and TKT-H (AC templates).
