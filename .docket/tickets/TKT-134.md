# TKT-134: Smart defaults by stack — auto-inject AC on ticket creation

## Description
When a ticket is created, detect the project's tech stack and inject default AC items with runnable verification commands. Stack detection reads package.json, pyproject.toml, go.mod, Cargo.toml, etc.

Built-in defaults:
- TypeScript/JS: tsc --noEmit, pnpm test, pnpm test --coverage
- Python: pytest, pytest --cov (>= 80%)
- Go: go test ./...
- Rust: cargo test

User can override/extend in .docket/config.yaml:
  ac_defaults:
    typescript:
      - desc: "No TypeScript errors"
        run: "pnpm tsc --noEmit"
      - desc: "Tests pass with coverage"
        run: "pnpm test --coverage"

Defaults are only injected if no ACs already exist on the ticket. Depends on TKT-133 (runnable AC commands).
