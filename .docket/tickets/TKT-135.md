# TKT-135: Human interactive AC prompt in pre-commit hook

## Description
When a human runs git commit and the commit references a docket ticket, the pre-commit hook renders the AC list interactively. For each incomplete AC item, it asks the user to confirm completion. Runnable ACs (TKT-133) are executed automatically; prose-only ACs get a y/n prompt.

Flow:
1. Hook extracts ticket ID from commit message
2. Fetches AC list via docket ac list TKT-NNN --format json
3. For each incomplete AC with run:: executes and shows pass/fail
4. For each incomplete AC without run:: prompts "Did you complete: <desc>? [y/N]"
5. Any 'n' or failed run blocks the commit with a clear message and fix suggestion

This is the same logic path as LLM auto-check — just different I/O. Both should call the same underlying library function.

Depends on TKT-131 (docket install), TKT-133 (runnable AC commands).
