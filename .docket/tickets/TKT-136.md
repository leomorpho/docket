# TKT-136: Scope locking — file claim tracking per ticket

## Description
When a ticket is moved to in-progress, docket begins tracking which files the work touches. File claims are recorded in .docket/locks.json. The pre-commit hook checks for overlapping claims between in-progress tickets and warns before allowing the commit.

docket worktree start TKT-NNN (new command) wraps git worktree add and registers the worktree + ticket in locks.json.

Locks are released when:
- Ticket moves to in-review or done
- docket worktree stop TKT-NNN is called
- Lock is manually cleared with docket lock release TKT-NNN

Lock detection sources: git diff --name-only from branch divergence point.

.docket/locks.json is gitignored (local-only state). Foundation for TKT-G (worktree conflict detection).
