# TKT-137: Worktree conflict detection + mutual exclusivity declarations

## Description
Allow users to declare relations between tickets that affect whether they can be safely worked on in parallel. Build on TKT-136 (scope locking) to provide explicit conflict detection before work begins.

New commands:
- docket link TKT-001 TKT-002 --relation blocks — TKT-001 blocks TKT-002
- docket link TKT-001 TKT-003 --relation parallel-safe — explicitly marked safe
- docket worktree start checks relations and file overlap before creating worktree

docket status (enhanced) shows a parallel work matrix: which in-progress tickets are safe to parallelize, which conflict.

Depends on TKT-136 (scope locking — file claim tracking).

## Acceptance Criteria
- [ ] docket link TKT-X TKT-Y --relation [blocks|parallel-safe|depends-on] persists the relation
- [ ] docket worktree start TKT-Y warns and requires --force if TKT-X blocks TKT-Y and is in-progress
- [ ] docket status --parallel renders a matrix of in-progress tickets with safe/risky indicators
- [ ] File overlap detection from TKT-136 feeds into the parallel matrix automatically
- [ ] Relations are shown in docket show TKT-NNN output
