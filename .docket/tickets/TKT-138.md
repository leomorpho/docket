# TKT-138: AC templates — named composable acceptance criteria sets

## Description
Ship named AC template bundles that can be applied at ticket creation time. Templates encode best-practice AC sets for common ticket archetypes. Built-in templates ship with docket; user templates live in .docket/templates/.

Built-in templates:
- api-endpoint: unit test, integration test, OpenAPI doc update, error handling, auth check
- react-component: unit test, visual regression, a11y check, storybook story
- db-migration: migration runs up/down cleanly, no data loss on rollback, tested on staging
- cli-command: help text present, happy path test, error path test, CLAUDE.md updated

docket create --ac-template api-endpoint injects the full template AC list (with run: commands where applicable).

User templates in .docket/templates/<name>.yaml override built-ins with same name. Depends on TKT-133 (runnable AC commands).

## Acceptance Criteria
- [x] docket create --ac-template <name> applies the named template's AC items to the new ticket : create supports --ac-template and applies named template AC
- [x] docket template list shows available built-in and user-defined templates : Added template list command for built-ins and user templates
- [x] User templates in .docket/templates/<name>.yaml override built-ins with the same name : User templates override built-ins by template name
- [x] Templates are composable: --ac-template api-endpoint,auth merges both template AC lists : Template composition supported via comma-separated names
- [x] docket template show <name> renders the AC list for a template before applying it : Added template show command to preview AC list
