# TKT-139: Session continuity checkpoints — auto-snapshot on AC completion

## Description
Auto-snapshot agent/session state each time an AC is marked complete, so work can be resumed by a new agent with full context. This addresses the most common LLM failure mode: context loss mid-task.

A checkpoint stores:
- Which ACs are done (with evidence)
- Files changed since ticket start (git diff summary)
- Last 3 comments on the ticket
- Current branch and worktree path

docket session resume TKT-NNN emits a structured context block that can be pasted or piped into a new agent session. Format is optimized for LLM consumption (not just human reading).

Also: docket session compress (already exists) gets a --checkpoint flag to save the compressed summary as a named checkpoint rather than just printing it.
