# TKT-140: docket ac update and ac remove commands

## Description
Currently AC items cannot be edited or deleted after creation. The only workaround is adding comments to clarify intent, which is fragile and pollutes the comment history.

Add two new subcommands:
- docket ac update TKT-NNN --step N --desc "new description" [--run "new command"] — update an existing AC item's description and/or run command by step number
- docket ac remove TKT-NNN --step N — remove an AC item by step number (with confirmation prompt unless --yes)

Step numbers are the 1-based index shown in docket ac list output.

Use case that triggered this: AC items are written before full design is settled and need correction as requirements clarify, without having to leave misleading text in place or bury corrections in comments.

## Acceptance Criteria
- [ ] docket ac update TKT-NNN --step N --desc "..." updates the description of the Nth AC item
- [ ] docket ac update TKT-NNN --step N --run "..." updates the run command of the Nth AC item
- [ ] docket ac remove TKT-NNN --step N removes the Nth AC item with a confirmation prompt
- [ ] docket ac remove TKT-NNN --step N --yes skips the confirmation prompt
- [ ] Step numbers in update/remove match the 1-based index shown in docket ac list output
