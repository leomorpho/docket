# TKT-143: Invisible Coordination: Trigger ticket claims on state transition

## Description
Make coordination "invisible" to LLM agents by automatically claiming tickets when they are transitioned to `in-progress`. This removes the need for agents to understand "Claiming" as a separate concept.

## Acceptance Criteria
- [ ] In the `update_ticket` MCP tool and CLI command:
    - If `state` transitions to `in-progress`, automatically call `internal/claim.Claim()`.
    - If `Claim()` fails (already claimed), return a 409-style error: "Cannot start work: TKT-001 is already claimed by Agent X in /other/path".
    - If `state` transitions to `done` or `archived`, automatically call `internal/claim.Release()`.
- [ ] Update `list_tickets` MCP tool to include a `claimed_by` field for information only.
