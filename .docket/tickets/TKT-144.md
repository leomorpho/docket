# TKT-144: Implement `docket install --skill` for Gemini CLI integration

## Description

Create a command to automatically generate and install a Docket "Skill" for the Gemini CLI. A Skill provides specialized domain knowledge and tool-use instructions directly to the agent. This will force agents to use Docket's MCP tools correctly and prevent them from attempting to modify the raw ticket files directly.

## Acceptance Criteria

- [x] Implement `docket install --skill` command.
- [x] Command should detect the presence of `gemini-cli` or default to `~/.gemini/skills/docket/`.
- [x] Automatically generate a `SKILL.md` file with the following content:
    - MCP server tool descriptions.
    - Explicit directive: "CRITICAL: Do not edit .docket/tickets/*.md directly. Use the 'docket' MCP tools for all modifications."
    - Instructions on ticket claiming (TKT-142/143).
    - Guidelines on using temporary files for large payloads (TKT-146).
- [x] Provide clear feedback on whether the skill was installed or if manual steps are needed.

## Technical Context

- A Gemini Skill is just a directory with a `SKILL.md` manifest.
- This represents the "one-click" experience to turn an LLM into a "Docket Agent."

## Plan

1. Define the content of `SKILL.md` for maximum agent effectiveness.
2. Implement the file writing logic in `cmd/install.go`.
3. Add checks for directory existence and permissions.
