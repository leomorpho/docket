# TKT-145: Expand docket install to support Cursor and VS Code

## Description
Evolve the `docket install` command to include automated or guided setup for Cursor (`.cursorrules`) and VS Code (`.vscode/settings.json`). This ensures that agents running in these IDEs are aware of the Docket MCP server and follow the same "No direct mutation" rules as the Gemini CLI agents.

## Acceptance Criteria
- [x] Implement `docket install --cursor`.
- [x] Command should append to or create a `.cursorrules` file with instructions for Cursor's internal agents to use the MCP server and avoid raw file edits.
- [x] Implement `docket install --vscode`.
- [x] Command should generate/update a `.vscode/settings.json` snippet with the MCP server configuration pointing to the local `docket serve --mcp` command.
- [x] Provide a summary to the user on what was modified and how to verify.
