# TKT-146: Optimize MCP tools for large payloads via temporary files

## Description
Address the "Permission Prompt" and shell argument length limits when LLMs pass massive descriptions or handoffs to Docket. Instead of forcing the LLM to pass thousands of characters in a single tool call, provide a mechanism for the agent to write the content to a temporary file first, which Docket then reads.

## Acceptance Criteria
- [ ] Update MCP tools `create_ticket` and `update_ticket` to accept an optional `content_file` (string) field.
- [ ] If `content_file` is provided, the MCP server must read the file and use its content as the ticket description/handoff.
- [ ] Implement a safe reading mechanism that validates the path (e.g., must be within the repo or a known temp dir).
- [ ] Update the Docket Skill (TKT-144) and Cursor rules (TKT-145) to instruct agents: "If your content is > 1000 characters, use a temporary file for the update."
