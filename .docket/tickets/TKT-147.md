# TKT-147: Force MCP Usage: Implement signature-based write protection

## Description

Force LLM agents to use the MCP server instead of editing ticket markdown files directly. We will implement a `write_hash` field in the ticket frontmatter that is generated only by the CLI/MCP. If an LLM attempts to use `sed` or `write_file` directly on the markdown, the hash will become invalid, and `docket validate` (pre-commit) will reject the change.

## Acceptance Criteria

- [ ] Add a hidden `write_hash` field to ticket YAML frontmatter.
- [ ] Every write via the `internal/store` package must update this hash (e.g., a HMAC or SHA-256 of the content minus the hash field).
- [ ] `docket validate` must check if the hash matches the content.
- [ ] Pre-commit hook (`lefthook`) must reject any staged ticket file with an invalid hash.
- [ ] Provide a clear error message: "Direct file mutation detected. You must use Docket's MCP tools to update tickets."

## Technical Context

This provides a hard enforcement layer. LLMs will quickly learn that direct edits are a waste of time because the build will fail.
