# TKT-148: Implement `docket fix` to repair ticket signatures and teach agents

## Description

Create the "Tutor" command, `docket fix`, which automates the repair of ticket files that have been directly edited by an LLM or human. Instead of just failing, Docket will now intelligently detect the intended change, apply it through the official storage logic (generating a valid `write_hash`), and then explicitly show the LLM the CLI command it *should* have used.

## Acceptance Criteria

- [ ] Implement `docket fix <TKT-ID>`.
- [ ] Command must:
    - Compare the current on-disk (invalid) ticket with the "last known good" version (e.g., from the Git index or a backup).
    - Detect the delta in fields (title, state, priority, etc.).
    - Apply these changes properly using `internal/store` to regenerate a valid `write_hash`.
    - Output a "Tutor Message": "Repairing TKT-148. Do not edit markdown files directly. Use the tools. You should have used: `docket update TKT-148 --field 'value'`."
- [ ] Implement `docket fix --all` to scan and repair all tickets.
- [ ] Command should be the primary suggestion when `docket validate` fails due to a signature mismatch.

## Technical Context

Requires TKT-147 (Write Protection Hash) to be implemented first. The goal is a self-healing system that actively teaches the LLM better habits.

## Plan

1. Develop a diffing utility to extract changes from an invalid ticket file.
2. Integrate with `internal/store` to re-persist and sign the file.
3. Design a template for the "Tutor Message" that is easy for agents to parse and follow.
