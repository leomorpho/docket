# TKT-149: Enhance pre-commit validation with 'Tutor' feedback

## Description
Integrate the `docket fix` "Tutor" logic into the pre-commit validation flow. This ensures that when an agent makes a mistake and tries to commit a manually edited ticket, Docket doesn't just block them but also provides a clear, actionable path to correction and learning.

## Acceptance Criteria
- [ ] Update `docket validate` output to detect signature mismatches (from TKT-147).
- [ ] When a mismatch is found, output a machine-readable suggestion: "🚨 Direct Mutation Detected. Run `docket fix TKT-NNN` to repair and see the correct usage."
- [ ] Update `lefthook.yml` (pre-commit) to ensure the full "Tutor Message" is visible to the agent during the commit process.
- [ ] Add a "Doctor" mode to `docket check` that summarizes all required repairs and system health issues.
