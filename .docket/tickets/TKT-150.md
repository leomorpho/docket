# TKT-150: Transparent Isolation: Automatically create Git worktrees for agents

## Description
Make parallelization the default experience by automatically isolating agent work into dedicated Git worktrees. When an agent transitions a ticket to `in-progress` via MCP, Docket will provision a clean environment for that ticket.

## Acceptance Criteria
- [ ] Modify `update_ticket` MCP tool handler:
- [ ] Automatically register the cross-worktree claim (TKT-142) for the newly created path.
- [ ] Ensure the temporary branch is based on the current `HEAD` of the primary branch.
