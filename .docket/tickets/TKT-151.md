# TKT-151: Transparent Integration: Merge-back and cleanup on ticket completion

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-151` before committing. -->

## Description
Complete the isolation lifecycle by automatically merging the isolated work back into the primary branch when a ticket is finished. This ensures the "master" branch always stays updated while keeping the work process isolated.

## Acceptance Criteria
- [ ] Transitioning a ticket to `done` or `archived` via MCP triggers:
    - `git merge` from the ticket branch into the original main branch.
    - On success: `git worktree remove` and branch deletion.
    - If a merge conflict occurs, the transition fails and provides the agent with a "Conflict Resolution Message" pointing to the worktree path.
- [ ] Implement an `internal/sync` package to handle the merge-back logic.
- [ ] Ensure that a claim (TKT-142) is released only *after* a successful merge.
