# TKT-153: Divergent Work Prevention: Force agents to check for existing tickets

## Description
Ensure that all development work flows through Docket by teaching LLM agents to always check for existing, similar tickets before starting any new task. This prevents 'Divergent Work' where an agent implements a feature or fix without a corresponding ticket, leading to a loss of context and auditability. The agent should search for related tickets, and if one is found, ask the user if they would like that ticket to be completed now.

- [ ] Improve the 'related' MCP tool to return a 'similarity_score'.
