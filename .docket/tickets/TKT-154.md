# TKT-154: Ecosystem Analytics: Track and visualize ticket lead time

## Description
To improve ticket specificity and understand agent performance, we need to track exactly when a ticket was 'started' (moved to in-progress) and 'completed' (moved to done). This data should then be visualized in the Svelte-based UI to show lead time and duration for each task.

## Acceptance Criteria
- [ ] Add 'started_at' and 'completed_at' fields to the ticket YAML frontmatter.
- [ ] Update the MCP and CLI state transition logic to automatically populate these timestamps.
- [ ] Update the Svelte UI (web/src/lib/types.ts and components) to display 'Duration' or 'Lead Time' for completed tickets.
- [ ] Implement an Analytics/Audit view in the UI to visualize task efficiency over time.
- [ ] Ensure that 'completed_at' is reset if a ticket is reopened.
- [ ] Go tests pass (run: go test ./...)
