# TKT-172: [Epic] Harden local trust, workflow governance, and agent execution

## Description
This umbrella epic tracks the next wave of Docket improvements discussed on 2026-03-13 around trust boundaries, workflow governance, worktree enforcement, model routing, context discipline, and documentation.

Use this epic as the parent for all implementation tickets in this initiative.

Context for implementers:
- Existing worktree automation tickets such as TKT-150, TKT-151, and TKT-157 are marked done, but current user experience suggests the intended behavior is either incomplete, bypassed, or not visible in normal flows.
- The repo currently treats ticket markdown as readable/editable source of truth. The new design must preserve good Git diffs and local-first DX while making privileged actions non-authoritative unless they pass through a trusted surface.
- The initial target is a single-user, local-first implementation with room to add OS-keystore and SaaS-backed adapters later.
- Tickets under this epic should stay atomic enough for a small LLM to complete without asking clarifying questions.

Out of scope for this epic:
- Full SaaS rollout.
- Full multi-user server-side trust management.
- Blockchain or decentralized consensus designs.

## Acceptance Criteria
- [ ] All child tickets under this epic are completed and validated in dependency order.
- [ ] The final design keeps workflow intent in the repo while keeping privileged authority outside normal agent reach.
- [ ] Repo behavior CLI behavior UI behavior and docs all reflect the same trust and workflow model.
