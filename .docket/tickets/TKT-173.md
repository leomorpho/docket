# TKT-173: [Epic] Build the local trust anchor and secure mode

## Description
This epic covers the local trust boundary for Docket.

Scope:
- Require and document a Docket home directory outside the repo.
- Introduce a local keystore contract and initial encrypted-file backend.
- Add a password-based secure mode for privileged actions.
- Namespace repo-specific trust anchors separately from global identity material.
- Prepare the design for later multi-device or remote key management without implementing SaaS now.

Why this exists:
Docket needs a trust root that is outside normal repo edits so privileged actions cannot be authorized only by changing markdown in the repository.

## Acceptance Criteria
- [ ] All child tickets under this epic are completed with a coherent local-first trust flow.
- [ ] A single-user developer can unlock privileged actions locally without a server or OS-specific secret store.
- [ ] The implementation keeps a clear adapter seam for future OS-keystore or SaaS custody.
