# TKT-174: [Epic] Govern workflow policy and privileged state transitions

## Description
This epic turns the workflow discussions into enforceable product behavior.

Scope:
- Separate editable workflow proposals from approved runtime policy.
- Introduce secure/admin activation for workflow changes.
- Split agent-safe and privileged capability surfaces.
- Enforce human-only closure semantics.
- Add structured workflow semantics and richer acceptance criteria for human verification.
- Move privileged UI mutations away from the generic CLI path.
- Make workflow and transition diffs intelligible to humans.

Out of scope:
- Full ledger implementation for every event (covered by the signed-state epic).

## Acceptance Criteria
- [ ] All child tickets under this epic are completed and Docket separates editable workflow proposals from approved runtime policy.
- [ ] Agents can continue normal work while privileged transitions and policy activation require the secure surface.
- [ ] Workflow configuration remains customizable without letting agents weaken active policy mid-run.
