# TKT-175: [Epic] Add repo-native signed state and rollback recovery

## Description
This epic introduces signed, replayable workflow state while keeping Git as the home of code history.

Scope:
- Define a signed event ledger format stored in the repo.
- Verify signatures and chain integrity when privileged operations run.
- Store the latest trusted head outside the repo to detect rollback attacks.
- Add repair/re-anchor tooling for damaged local trust state.

Out of scope:
- Replacing Git diffs with ledger diffs.
- Storing code diffs or blobs in Docket state.

## Acceptance Criteria
- [ ] All child tickets under this epic are completed and Docket can detect forged or rolled-back privileged state.
- [ ] Repo state stays git-friendly while privileged trust decisions rely on verifiable signatures and external anchors.
- [ ] Recovery and repair flows let a user re-establish trust without losing repo history.
