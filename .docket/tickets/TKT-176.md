# TKT-176: [Epic] Enforce agent worktrees and execution traceability

## Description
This epic covers the execution path for agent work.

Scope:
- Audit why previously completed worktree tickets are not matching expected behavior in practice.
- Make worktree creation mandatory for agent-managed runs.
- Bind runtime metadata to each run so Docket knows which ticket, worktree, and workflow hash are active.
- Enforce ticket-linked commits for Docket-managed runs.
- Add Docket-native enforcement hooks for start/review/QA/privileged gates.

## Acceptance Criteria
- [ ] All child tickets under this epic are completed and agent-managed runs occur inside Docket-created worktrees only.
- [ ] Each Docket-managed run can be traced through run metadata commit linkage and enforcement hooks.
- [ ] The implementation explains and resolves the gap between prior worktree tickets and current observed behavior.
