# TKT-177: [Epic] Improve AI orchestration, context discipline, and docs

## Description
This epic gathers the orchestration, memory, and documentation improvements from the design discussion.

Scope:
- Route work by capability tier instead of hardcoded provider model names.
- Reset and resume context cleanly between ticket runs.
- Generate Docket-managed skill/instruction packs from approved policy.
- Capture repeated human corrections as scoped learnings.
- Write extensive docs that explain the current trust model and future evolution path.
- Evaluate whether modes are sufficient or specialized agents are worth the extra complexity.

## Acceptance Criteria
- [ ] All child tickets under this epic are completed and Docket provides repeatable AI workflow guidance instead of ad hoc prompts.
- [ ] Model selection context resets run resumes and lightweight learnings are captured in product behavior or docs.
- [ ] The outcome remains simple enough to explain to a small LLM through generated instructions and documentation.
