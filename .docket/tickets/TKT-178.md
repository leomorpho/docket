# TKT-178: Require DOCKET_HOME and define the Docket home contract

## Description
Implement the baseline storage contract for the new trust model.

Context:
- The current repo stores `.docket/` data inside the repo root, but the new security design needs a second storage area outside the repo.
- The user explicitly wants `DOCKET_HOME` to be required and clearly communicated; do not silently fall back to OS defaults in this first version.

What to do:
- Introduce one canonical resolver for `DOCKET_HOME`.
- Use that resolver early enough that the tool fails before trying privileged work with an ambiguous storage location.
- Make the error message explain why this variable exists and show an example path.
- Update docs/help/install messaging accordingly.

Likely areas:
- `cmd/root.go`
- `cmd/install.go`
- future security packages under `internal/`

Out of scope:
- Implementing the keystore itself.

## Acceptance Criteria
- [ ] CLI startup fails fast with a clear actionable message when DOCKET_HOME is missing malformed or unusable.
- [ ] Install init help output and docs explain that secure features depend on DOCKET_HOME and show setup examples.
- [ ] Automated tests cover environment resolution and the failure path.
