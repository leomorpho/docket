# TKT-181: Namespace per-repo trust anchors under stable repo IDs in DOCKET_HOME

## Description
Separate shared identity from repo-specific anchors.

Context:
- The same Docket installation should work across multiple repos without mixing their trust heads together.
- The repo-local `.docket/` directory can hold a stable repo ID file, but the trusted anchors themselves must live under `DOCKET_HOME`.

What to do:
- Define the repo ID format and where it is stored in the repo.
- Add lookup and initialization logic for repo namespaces inside `DOCKET_HOME`.
- Store repo-specific trust anchor files separately from the global keystore.

Likely areas:
- `.docket/` repo metadata
- new security/path helper packages

Out of scope:
- Full recovery tooling (covered separately).

## Acceptance Criteria
- [ ] Each repo receives a stable Docket repo identifier and a dedicated namespace under DOCKET_HOME for trust anchors and runtime metadata.
- [ ] Global identity material stays shared per user or device while repo-specific data remains isolated per repo.
- [ ] Repo moves or renames do not orphan trust state because lookup uses the stable repo ID rather than only the filesystem path.
