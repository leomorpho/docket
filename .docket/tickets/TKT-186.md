# TKT-186: Enforce human-only done and other privileged state transitions

## Description
Make closure semantics enforceable instead of instructional.

Context:
- The product rule is that agents can implement work and move tickets to review-like states but humans own closure.
- Do not rely on UI-only restrictions; enforcement has to live in Docket runtime behavior.

What to do:
- Define which transitions are privileged in code.
- Reject those transitions from the agent-safe surface.
- Ensure the UI and any server endpoints use the privileged path when closure is attempted.

Dependencies to read first:
- capability-surface split
- workflow lock and activation model

## Acceptance Criteria
- [x] Agent-safe mutation paths cannot move a ticket into done or another privileged terminal state. : CLI agent-safe mutation guard + tests
- [x] Privileged transitions succeed only through the secure or admin surface with explicit human confirmation. : secure surface required with explicit confirmation
- [x] Tests cover rejection from agent-safe paths and success from secure or admin paths across CLI and UI entry points. : CLI/UI rejection and secure success tests added
