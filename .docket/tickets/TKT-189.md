# TKT-189: Move privileged UI mutations onto the secure or admin surface

## Description
Align the UI with the new capability boundary.

Context:
- The current UI shells out through the CLI layer for ticket mutations.
- That architecture makes it too easy for UI-driven and agent-driven mutations to share the same authority path.

What to do:
- Audit the web routes under web src routes api and web src lib server.
- Move privileged mutations to the secure or admin surface.
- Show explicit secure-mode state in the UI so the user understands when they are approving privileged work.

Out of scope:
- Full secure-mode visualization polish beyond what is needed to make the flow clear.

## Acceptance Criteria
- [ ] The web UI no longer performs privileged ticket mutations through the same generic CLI path used for ordinary changes.
- [ ] Privileged UI actions require secure mode and make the approval step visible before the mutation is attempted.
- [ ] Server-side and end-to-end tests cover a privileged mutation path and prove the non-privileged path can no longer perform it.
