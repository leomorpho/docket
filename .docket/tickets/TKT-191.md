# TKT-191: Define a signed event ledger format and append rules for privileged runtime events

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-191` before committing. -->

## Description
Define the repo-native signed state model.

Context:
- The repo should keep readable reviewable event history for ticket and runtime decisions while Git continues to own code history.
- The event ledger does not need to contain code diffs; it should reference commits and runtime metadata instead.

What to do:
- Define event types for at least workflow activation privileged transition approval run start or stop and commit linkage.
- Include actor identity timestamp workflow hash and previous-event hash.
- Keep the format Git-reviewable and stable.

Out of scope:
- Rollback detection anchor.

## Acceptance Criteria
- [ ] Docket has a versioned event format for privileged ticket and runtime events stored in the repo and linked by previous-event hash.
- [ ] The event schema records enough information to reconstruct privileged workflow history without storing source diffs that belong in Git.
- [x] Tests cover serialization hash chaining and rejection of malformed events or impossible predecessor references. : Added tests for append+serialization, hash chaining, malformed events, and impossible predecessor references in internal/security/event_ledger_test.go.
