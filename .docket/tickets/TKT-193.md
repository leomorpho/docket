# TKT-193: Store the trusted ledger head outside the repo and detect rollback

## Description
Close the rollback loophole in the signed-state design.

Context:
- Signatures stop forgery but do not stop a writable workspace from replacing current repo state with an older valid version.
- The minimal external trust anchor for the first version is the latest trusted head stored outside the repo.

What to do:
- Persist the trusted head in the repo namespace under DOCKET_HOME.
- Compare repo state against that anchor during privileged flows.
- Produce errors that explain what happened and how to repair it.

Dependencies to read first:
- repo namespace design
- event ledger format

## Acceptance Criteria
- [ ] Docket stores the latest trusted ledger head or equivalent monotonic anchor outside the repo under the repo namespace in DOCKET_HOME.
- [ ] When the repo contains an older but otherwise valid ledger chain Docket detects the rollback and refuses to treat it as the current trusted state without explicit repair.
- [ ] Tests cover normal head advancement stale or rolled-back repo state detection and behavior when the external anchor is missing or corrupted.
