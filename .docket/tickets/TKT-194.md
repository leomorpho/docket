# TKT-194: Build repair and re-anchor tooling from Git history and valid signed events

## Description
Add an operator-grade recovery path for the local-first trust model.

Context:
- If the keystore or trusted head is damaged the repo data still exists but the user needs a supported way to re-establish trust.
- Recovery should be explicit inspectable and consistent with the local-first story.

What to do:
- Add commands or flows to inspect candidate ledger heads.
- Let the user choose or confirm the head that should become trusted.
- Rebuild local anchor state without modifying Git history.

Out of scope:
- Hosted recovery services.

## Acceptance Criteria
- [ ] A user can inspect validate and repair broken local trust anchors by rebuilding state from Git history plus valid signed repo events.
- [ ] The repair flow never silently rewrites repo history and it explains which head or anchor the user is choosing to trust.
- [ ] Tests cover a corrupted-anchor case and a missing-anchor case.
