# TKT-196: Require Docket-created worktrees for agent-managed runs and fail closed without fallback

## Description
Turn worktree usage into system behavior rather than guidance.

Context:
- The user wants Docket-managed ticket work to always happen in a git worktree.
- Existing worktree tickets already laid some groundwork; this ticket should build on that work after the audit ticket identifies the actual gap.

What to do:
- Bind agent start and resume flows to Docket worktree creation or lookup.
- Remove or reject fallback paths that continue in the main checkout.
- Keep the human workflow outside Docket-managed runs unconstrained.

## Acceptance Criteria
- [x] Agent-managed ticket runs cannot start unless Docket creates or verifies the dedicated worktree for that run. : WorkflowManager.StartTask now enforces dedicated worktree binding for agent:* actors
- [x] If worktree creation fails Docket returns a clear error and does not silently fall back to the main checkout or another arbitrary directory. : agent worktree-dir or creation failure now returns explicit errors and blocks start
- [x] Tests cover successful worktree creation expected failure modes and rejection of agent-managed runs outside the bound worktree. : added workflow + session resume tests for success/failure and out-of-worktree rejection
