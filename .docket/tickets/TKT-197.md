# TKT-197: Add run manifests that bind ticket actor worktree and workflow hash

## Description
Add explicit runtime identity for agent-managed work.

Context:
- Later gates such as commit linkage context reset and secure review need a durable record of what run is active and what workflow hash it was pinned to.
- Prefer a small explicit manifest over implicit assumptions spread across multiple packages.

What to do:
- Define the manifest format and storage location.
- Record it when a Docket-managed run starts.
- Make follow-up commands able to verify the current runtime context against it.

## Acceptance Criteria
- [x] Every Docket-managed run records a manifest tying together the ticket ID actor identity or type worktree path branch and active workflow lock hash. : run manifests now persist ticket/actor/type/worktree/branch/workflow hash on run start
- [ ] Run manifests are stable enough for resume audit and cross-surface checks without storing source diffs or other data that belongs in Git.
- [ ] Tests cover manifest creation lookup stale-manifest handling and rejection when runtime context does not match the stored manifest.
