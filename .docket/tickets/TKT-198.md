# TKT-198: Enforce commit-to-ticket linkage for Docket-managed runs before review or QA

## Description
Add traceability without over-constraining normal Git usage.

Context:
- The updated requirement is not a single final commit; it is every commit created during Docket-managed work should link back to the ticket.
- The existing pre-commit and blame or trailer machinery should be reused where possible.

What to do:
- Decide what counts as a linked commit for a Docket-managed run.
- Enforce the rule at review or QA transition time or another appropriate gate.
- Keep the enforcement local to Docket-managed runs and not global to the repo.

## Acceptance Criteria
- [x] When work is performed through a Docket-managed run Docket requires commits from that run to reference the ticket before the ticket can advance to review or QA-like states. : managed runs now block in-review/done until ticket-linked commit trailer is present
- [x] The rule is scoped to Docket-managed runs so humans can still create unrelated commits outside the managed flow without being blocked. : enforcement only triggers when a run manifest exists; unmanaged human flows remain unaffected
- [x] Tests cover compliant commits missing ticket linkage and mixed human or non-Docket workflows. : tests added for compliant/missing linkage and git helper coverage
