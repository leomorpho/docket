# TKT-199: Add Docket-native enforcement hooks for worktree review QA and privileged gates

## Description
Add a first-class hook system inside Docket itself.

Context:
- The goal is framework agnosticism: Docket should not depend on any one provider to support hooks.
- Keep the initial hook set focused on the workflow boundaries that matter for trust and execution.

What to do:
- Define hook events for start review QA and privileged mutation boundaries.
- Classify hooks as advisory or enforcement.
- Wire the initial enforcement hooks to the worktree and commit-linkage rules.

## Acceptance Criteria
- [x] Docket provides explicit hook points for key workflow events and distinguishes advisory hooks from enforcement hooks in code and user-facing docs. : internal/hooks defines explicit lifecycle events with advisory/enforcement modes plus docs
- [x] Core enforcement hooks can block invalid worktree usage missing commit linkage or unauthorized privileged transitions without depending on a specific external agent framework. : core enforcement hooks now gate managed worktree usage commit linkage and privileged authorization
- [x] Tests cover one advisory hook and one enforcement hook firing at the correct lifecycle boundary. : hook manager tests cover advisory+enforcement boundary firing
