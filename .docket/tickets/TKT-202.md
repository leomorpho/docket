# TKT-202: Reset context on ticket worktree and run boundaries and persist structured resume artifacts

## Description
Make context discipline a product feature rather than an informal habit.

Context:
- Ticket changes worktree changes and run changes are deterministic points where stale context should be discarded.
- The existing session and handoff functionality should be reused or extended where practical.

What to do:
- Define reset triggers.
- Persist a compact resume artifact rather than replaying whole transcripts.
- Make the data easy for later agent instructions and human review to consume.

## Acceptance Criteria
- [ ] Docket resets active AI context when the ticket worktree or run identity changes instead of carrying stale context across unrelated work.
- [ ] Each run produces a structured resume artifact with summary changed files linked commits blockers and next steps so another tool or model can pick up the work later.
- [ ] Tests cover context-reset triggers and serialization of the structured resume data.
