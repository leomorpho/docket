# TKT-203: Generate Docket-managed instruction packs from approved workflow policy

## Description
Distribute Docket workflow rules consistently across agent environments.

Context:
- Skills and instruction packs are useful for alignment but enforcement must still live inside Docket.
- Keep the first version focused on generated artifacts and versioning rather than trying to support every framework feature on day one.

What to do:
- Define an internal instruction-pack representation.
- Generate it from the approved workflow lock and relevant routing or context settings.
- Integrate it with existing install or help-json surfaces where appropriate.

## Acceptance Criteria
- [ ] Docket can generate or install a machine-readable instruction pack derived from the approved workflow lock so supported agent frameworks receive consistent guidance.
- [ ] Instruction packs are versioned or otherwise pinned to the workflow policy they were generated from.
- [ ] Tests cover at least one generated pack format and prove the output changes when relevant workflow policy changes.
