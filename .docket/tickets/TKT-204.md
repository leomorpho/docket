# TKT-204: Phase 2: Add human-approval and scoped-replay controls atop baseline learning engine

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-204` before committing. -->

## Description
Context: Baseline learning capture and replay now live in newer backlog items (TKT-223 and TKT-224). This ticket should avoid duplicating baseline ingestion/ranking work and instead add governance controls required for higher-trust workflows.

Objective: Add explicit human-approval and scope-governance semantics on top of the baseline learning engine.

Scope:
- Add proposal workflow so captured learn rules are inactive until approved.
- Add scoped activation and supersession rules so learnings apply only where intended.
- Add auditability for proposal, approval, rejection, and supersession events.

Out of scope:
- Re-implementing baseline learning capture storage.
- Re-implementing baseline replay ranking in start flows.

Delivery notes: Implement only the delta from baseline and preserve compatibility with existing learning records where feasible.

## Acceptance Criteria
- [ ] Docket can turn repeated human corrections into proposed learnings that require explicit approval before they become active guidance.
- [ ] Approved learnings carry scope metadata so only relevant learnings are replayed for a given ticket subsystem or workflow state.
- [ ] Tests cover proposal creation approval or rejection and scoped retrieval of relevant learnings.
