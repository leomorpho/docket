# TKT-204: Add lightweight learning capture with human approval and scoped replay

## Description
Capture recurring corrections without turning memory into an unbounded dump.

Context:
- The product should learn from repeated mistakes but only after a human explicitly approves the generalized rule.
- Learnings should be lightweight scoped and easy to supersede later.

What to do:
- Define the learning record structure.
- Add a way to propose a learning from a correction.
- Surface relevant learnings when a new ticket run starts.

## Acceptance Criteria
- [ ] Docket can turn repeated human corrections into proposed learnings that require explicit approval before they become active guidance.
- [ ] Approved learnings carry scope metadata so only relevant learnings are replayed for a given ticket subsystem or workflow state.
- [ ] Tests cover proposal creation approval or rejection and scoped retrieval of relevant learnings.
