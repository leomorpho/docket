# TKT-206: Evaluate modes versus specialized agents after hooks and instruction packs land

## Description
Defer multi-agent complexity until the lower-level workflow primitives are in place.

Context:
- The design discussion identified real value in specialized agents but also a real risk of chaos if agents hooks skills and state machines all arrive at once.
- By the time this ticket starts Docket should already have hooks model routing tiers context resumes and generated instruction packs to evaluate against.

What to do:
- Compare a mode-based design against a specialized-agent design.
- Use Docket actual architecture and DX constraints in the analysis.
- Produce a decision record and follow-up recommendations.

## Acceptance Criteria
- [ ] Produce a written comparison of a single runtime with modes versus multiple specialized agents using Docket architecture as the baseline.
- [ ] The analysis makes a concrete recommendation for the next product step rather than stopping at generic pros and cons.
- [ ] The evaluation does not implement a full multi-agent framework and only creates the decision record plus any follow-up tickets needed.
