# TKT-207: Spike: design a ticket-scope security audit hook and artifact

## Description
Evaluate whether Docket should add a ticket-scope security audit hook inspired by the pre-ship prompt pattern of auditing for exposed secrets missing auth checks unprotected routes and unsanitized inputs.

This is intentionally a spike only.

Goal:
- Decide whether Docket should support a ticket-scope security audit artifact and hook.
- Define the smallest good design if the answer is yes.
- Avoid implementing the actual hook until the surrounding workflow primitives are in place.

What to evaluate:
- When the audit should run such as pre-review or pre-ready-for-qa.
- Which tickets should trigger it automatically based on labels changed files or sensitive areas like auth payments user data uploads or webhooks.
- What the output artifact should look like and where it should live.
- Whether the first version should be advisory only or block on unresolved high-severity findings.
- Whether full-repo audits should be a separate release-time workflow instead of part of normal ticket execution.
- Whether Docket should use a pure LLM audit prompt or combine deterministic checks such as secret scanning auth coverage checks and schema validation checks.

Deliverable:
- A written recommendation that fits the current Docket roadmap and explicitly references the hooks acceptance-criteria resume-artifact and instruction-pack work that would need to exist first.

Out of scope:
- Implementing the hook.
- Implementing the artifact.
- Running actual security scans as part of normal ticket flow.

## Acceptance Criteria
- [ ] Produce a written proposal for a ticket-scope security audit flow that defines trigger rules artifact format model tier guidance and advisory versus blocking behavior.
- [ ] Compare LLM-only auditing against a hybrid approach that combines deterministic checks with LLM reasoning and make a concrete recommendation for Docket.
- [ ] The spike does not implement the hook and instead ends with clear follow-up recommendations or new tickets if needed.
