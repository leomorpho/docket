# TKT-208: Fix docket ui project registration 404s

## Description
`docket ui` can start the Vite server, but then repeatedly fails to register the current project and the UI effectively dies from the caller's perspective.

Observed output:

```text
VITE v7.3.1  ready in 1074 ms

➜  Local:   http://127.0.0.1:43173/
➜  press h + enter to show help

[404] POST /api/projects/register
[404] POST /api/projects/register
[404] POST /api/projects/register
[404] POST /api/projects/register
[404] POST /api/projects/register
```

Relevant code paths suggest an endpoint mismatch:
- `cmd/ui.go` posts project registration to `/api/projects/register`
- `web/src/routes/api/projects/+server.ts` handles `POST /api/projects`

This ticket should make `docket ui` register the project successfully instead of spamming 404s when the UI server is already running or when the browser-open retry path attempts registration.

## Acceptance Criteria
- [ ] docket ui registers the current project without POSTing to a missing endpoint or spamming 404s
- [ ] Both the already-running UI path and the startup retry path use a valid project registration route
- [ ] Regression coverage exercises the CLI registration flow against the route exposed by the UI server (run: go test ./cmd/...)
