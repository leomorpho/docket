# TKT-209: Epic: Cross-platform agent capability foundation for Codex, Claude Code, and Gemini

## Description
Context: Docket already has robust ticket primitives, but AI harness integrations are inconsistent. MCP, skill instructions, and hook behavior are fragmented, and there is no single reliability contract proving setup health across platforms.

Objective: Deliver a simple v1 that is non-blocking by default and works consistently for Codex, Claude Code, and Gemini through one adapter architecture.

Scope:
- Canonical capabilities contract for workflow, hooks, and skills.
- Adapter-driven bootstrap and doctor surfaces.
- Start output that always shows integration readiness and remediation.
- Learning capture and replay loop to improve future runs.

Success criteria: Child backlog lands with deterministic tests, integration harness coverage, and clear validation evidence captured in tickets and comments so future adapters can be added without redesign.

## Acceptance Criteria
- [ ] All child tickets are linked to this epic with parent relationships and explicit dependency intent.
- [ ] Each child ticket includes unit-test, integration-test, and validation-evidence acceptance criteria.
- [ ] Epic progress is tracked via comments capturing milestone completion and final end-to-end validation evidence.
