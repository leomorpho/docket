# TKT-210: Define capabilities contract v1 for workflow, hooks, and skills

## Description
Context: Cross-platform behavior needs one canonical contract so adapter outputs and runtime behavior cannot drift.

Objective: Define a versioned capabilities contract and deterministic hash strategy used by CLI and MCP outputs.

Scope:
- Typed schema and model for workflow phases, hook events, and skill inventory.
- Contract versioning and hash generation semantics.
- Runtime write path at .docket/runtime/capabilities.json.
- Backward compatibility notes for future schema evolution.

Out of scope: Adapter-specific rendering details.

Delivery notes: Contract changes must be deterministic and easy to diff and review.

## Acceptance Criteria
- [ ] Unit tests verify required fields, schema validation errors, and deterministic contract hashing for unchanged payloads.
- [ ] Integration test writes and reloads capabilities contract in a temp repo and confirms roundtrip parity.
- [ ] Validation evidence includes contract version and hash emitted in test output and summarized in a ticket comment.
