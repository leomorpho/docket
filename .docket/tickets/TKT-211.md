# TKT-211: Implement adapter interface and registry for Codex, Claude Code, and Gemini

## Description
Context: Platform support should expand through adapters, not core flow branching.

Objective: Introduce a stable adapter contract and registry factory so bootstrap, start, and doctor can run through one abstraction.

Scope:
- Adapter interface for bootstrap, doctor checks, status, and install surfaces.
- Registry and factory with deterministic lookup and unsupported fallback behavior.
- Adapter metadata model consumed by command surfaces.

Out of scope: Concrete adapter implementations beyond stub wiring.

Delivery notes: Error messages should be explicit when adapters are missing or incomplete.

## Acceptance Criteria
- [ ] Unit tests cover registry lookup, unsupported-adapter fallback, and interface-level error propagation.
- [ ] Integration test simulates adapter selection in temp repos and verifies resolved adapter metadata.
- [ ] Validation evidence captures adapter resolution matrix output and is attached to ticket comments.
