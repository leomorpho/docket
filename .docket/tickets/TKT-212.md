# TKT-212: Add adapter auto-detection with explicit override and ambiguity handling

## Description
Context: Most users will not manually choose adapters each run, but detection must remain predictable.

Objective: Add deterministic adapter detection for Codex, Claude Code, and Gemini with explicit override support.

Scope:
- Environment and config heuristics for three target harnesses.
- Override flag that always wins over auto-detect.
- Clear warning and fallback behavior when detection is ambiguous.

Out of scope: Heuristics for every future harness.

Delivery notes: Detection behavior must be documented and testable without external services.

## Acceptance Criteria
- [x] Unit tests cover positive and negative detection cases for each target adapter. (run: go test ./internal/adapters -run TestRegistryResolveWithInfoEnvAndConfigHeuristics|TestRegistryResolveAutoDetectDeterministicOrder|TestRegistryResolveWithInfoInvalidHints) : Extended registry tests with positive/negative env/config/marker detection cases for codex, claude-code, and gemini plus invalid hint handling.
- [x] Integration test verifies override behavior supersedes auto-detection and is reflected in command output. (run: go test ./cmd -run TestBootstrapCommandOverrideReflectedInJSONOutput|TestBootstrapCommandAmbiguityWarningInHumanOutput) : Added bootstrap integration tests proving explicit override supersedes auto-detect and appears in command output with adapter_source.
