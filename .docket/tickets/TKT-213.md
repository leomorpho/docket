# TKT-213: Create idempotent docket bootstrap command with cross-link to docket start

## Description
Context: Setup drift is the top reason agents do not use MCP and skills reliably.

Objective: Ship docket bootstrap as the single idempotent setup entrypoint for installing MCP, skills, and hooks through adapters.

Scope:
- New command orchestration for adapter install steps.
- Per-step status reporting with clear changed and no-change results.
- Idempotent rerun behavior.
- Output ends with an instruction to run docket start.

Out of scope: Complex one-time migration flows outside adapter ownership.

Delivery notes: Bootstrap should be safe in existing repos and never require interactive prompts.

## Acceptance Criteria
- [x] Unit tests verify bootstrap orchestration order, step-level error bubbling, and idempotent rerun behavior. : go test ./cmd -run 'TestExecuteBootstrapOrchestrationOrder|TestExecuteBootstrapStepErrorBubbles|TestExecuteBootstrapIdempotentRerunBehavior'
- [x] Integration test runs bootstrap twice in temp repos and asserts second pass is no-op-safe. : go test -v ./cmd -run TestBootstrapCommandRunsTwiceNoOpSafe
- [x] Validation evidence includes before/after artifact inventory and bootstrap summary output recorded in ticket comments. : Bootstrap summary and before/after artifact inventory captured from verbose integration test logs
