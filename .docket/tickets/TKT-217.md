# TKT-217: Upgrade docket start with capability digest, status strip, and bootstrap remediation

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-217` before committing. -->

## Description
Context: Start is the primary entrypoint and must show integration health without overwhelming users.

Objective: Add concise capability digest and readiness strip to docket start output across human and JSON formats.

Scope:
- Render five-phase flow summary.
- Show MCP, Skills, and Hooks readiness for active adapter.
- Include one-line remediation pointing to docket bootstrap when needed.
- Keep output compact and action-first.

Out of scope: Verbose README-style capability dumps in start output.

Delivery notes: Output must remain non-blocking and deterministic.
