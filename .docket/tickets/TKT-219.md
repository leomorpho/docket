# TKT-219: Implement docket doctor health checks for MCP, skills, hooks, and contract state

## Description
Context: Without health checks, configuration drift remains invisible and agents silently skip integrations.

Objective: Add doctor command with PASS and FAIL checks and direct remediation commands.

Scope:
- Human-readable one-page status.
- JSON output for automation and CI.
- Checks for MCP wiring or reachability, skills install, hooks wiring, and contract presence.

Out of scope: Long-term analytics dashboards.

Delivery notes: Failure output must be actionable and concise.

## Acceptance Criteria
- [ ] Unit tests verify check state transitions and remediation command formatting for each failure mode.
- [ ] Integration test runs doctor before and after bootstrap and asserts expected status changes.
- [ ] Validation evidence includes doctor JSON artifacts archived by integration tests and linked in comments.
