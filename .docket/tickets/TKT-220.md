# TKT-220: Eliminate interactive AC or hook prompts and enforce deterministic non-blocking flow

## Description
Context: Docket should never block on interactive prompts during normal agent operation.

Objective: Remove prompt-based AC or hook behavior and replace it with deterministic pass or fail outcomes plus remediation text.

Scope:
- Remove interactive confirmation reads in AC and hook paths.
- Preserve invariant checks while keeping runtime non-interactive.
- Keep hooks advisory by default unless explicit enforcement is configured.

Out of scope: Removing all enforcement checks for hard invariants.

Delivery notes: Behavior must be stable across CI and local non-interactive shells.

## Acceptance Criteria
- [ ] Unit tests verify AC and hook code paths are non-interactive and deterministic for identical inputs.
- [ ] Integration test simulates non-interactive execution and confirms no prompt waiting occurs.
- [ ] Validation evidence captures deterministic stderr or stdout remediation traces in test artifacts.
