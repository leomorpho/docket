# TKT-221: Implement minimal lifecycle event set with stable payload schema

## Description
Context: A small stable event vocabulary is needed for observability and future extension without complexity.

Objective: Emit minimal lifecycle events with versioned payloads that adapters and tests can rely on.

Scope:
- Add run.start, phase.end, run.end, and tool.failure events.
- Define stable payload fields and versioning.
- Integrate event emission into relevant command flows.

Out of scope: Large event taxonomy or complex routing.

Delivery notes: Event ordering should be deterministic in integration harness runs.

## Acceptance Criteria
- [ ] Unit tests validate payload schema and required fields for each emitted event type.
- [ ] Integration test executes a simulated run and asserts expected event order and payload presence.
- [ ] Validation evidence includes event log fixtures and schema-validation results referenced in comments.
