# TKT-222: Generate minimal cross-platform skill pack v1 from canonical contract

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-222` before committing. -->

## Description
Context: Skills should be maintained once and rendered per adapter to avoid divergence.

Objective: Implement minimal skill pack generation from the shared capabilities contract.

Scope:
- Shared skill metadata and concise instruction model.
- Adapter-specific rendering and install artifacts.
- Drift checks ensuring generated output matches source contract.

Out of scope: Large catalog or advanced authoring UI.

Delivery notes: Skill content should remain concise and action-oriented for agent consumption.

## Acceptance Criteria
- [x] Unit tests verify skill metadata validation and renderer output consistency for each adapter. : go test ./internal/skills -run "TestBuildPackValidationAndRendererConsistency|TestBuildPackRejectsInvalidMetadata" && go test ./internal/adapters -run "TestCodexInstallMergesConfigWithoutClobbering|TestClaudeAdapterInstallMergesMCPConfig|TestGeminiInstallIdempotentAndCreatesArtifacts"
- [x] Integration test bootstraps each adapter and confirms installed skill IDs align with canonical contract. : go test -v ./cmd -run TestBootstrapAdapterSkillIDsAlignWithCanonicalContract (bootstraps codex/claude-code/gemini and validates installed skill IDs against canonical runtime contract)
- [x] Validation evidence captures contract-to-skill mapping report and artifact checksum snapshots. : Integration test writes per-adapter contract-to-skill mapping reports and checksum snapshots under .docket/test-fixtures/skills/{adapter}/mapping-report.json and checksum.txt; fixture paths emitted in verbose test logs.
