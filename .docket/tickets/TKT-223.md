# TKT-223: Implement learning capture parser and storage for LEARN rules

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-223` before committing. -->

## Description
Context: A lightweight learning loop is needed to compound improvements across sessions.

Objective: Parse and store learn entries from session or comment artifacts using the LEARN rule format.

Scope:
- Parser for LEARN lines with category and rule text.
- Durable local storage with timestamps and dedupe behavior.
- Ingestion from session and comment sources.

Out of scope: Heavy analytics or visualization features.

Delivery notes: Parser should be resilient to noisy surrounding text.

- [x] Integration test ingests synthetic session or comment artifacts and verifies expected learn entries are captured. : go test -v ./cmd -run TestLearningCaptureFromSessionAndCommentArtifacts (ingests synthetic session + comment artifacts and verifies captured LEARN entries)
