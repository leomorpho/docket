# TKT-224: Add learning replay in docket start with top 3 relevant rules

## Description
Context: Captured learnings only add value if they are surfaced when work begins.

Objective: Replay top relevant learn rules in start context using ticket text and metadata.

Scope:
- Relevance ranking with deterministic tie-breaking.
- Start output section showing top three concise rules.
- Graceful no-data behavior with no blocking.

Out of scope: Expensive semantic infrastructure for v1.

Delivery notes: Replay output must stay short and avoid overwhelming start UX.
