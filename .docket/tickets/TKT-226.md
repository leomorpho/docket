# TKT-226: Create fake-repo integration harness simulating LLM goal-driven runs

## Description
Context: We cannot prove model cognition, but we can validate deterministic behavior and emitted artifacts in simulated runs.

Objective: Build a fake-repo integration harness that simulates lifecycle execution and validates contract-driven outcomes.

Scope:
- Temp repo setup helpers with seeded tickets and config.
- Command-driven lifecycle simulation for non-blocking flow.
- Assertions for transitions, event traces, manifest outputs, and health checks.

Out of scope: Live external harness API integration.

Delivery notes: Harness should be reusable for future adapter expansion.

## Acceptance Criteria
- [ ] Unit tests cover harness setup helpers, fixture seeding, and command execution wrappers.
- [ ] Integration tests include happy-path and failure-retry scenarios with deterministic artifact assertions.
- [ ] Validation evidence stores run manifests and event traces as fixtures referenced in ticket comments.
