# TKT-228: Epic: LLM-friendly ticket authoring and automation surface

## Description
Context: LLMs currently struggle to author high-quality Docket tickets reliably because creation requires many sequential CLI calls, shell quoting is fragile, and errors are not always machine-actionable.

Objective: Add a simple, transactional ticket authoring surface that works consistently across Codex, Claude Code, and Gemini with non-interactive behavior by default.

Scope:
- Atomic spec-based ticket and backlog apply commands.
- Automation-safe runtime mode.
- Machine-readable error envelopes and guidance.
- Scaffolding tools and docs for LLM-first usage.
- Deterministic test coverage for happy-path and failure-path behavior.

Success criteria: An LLM can create groomed epic-plus-children backlogs via one or two commands with clear JSON feedback and minimal retries.
