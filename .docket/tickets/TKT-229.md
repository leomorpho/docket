# TKT-229: Define ticket and backlog apply spec schema with validator

## Description
Context: We need one canonical spec format before adding transactional apply commands.

Objective: Define and validate ticket and backlog apply spec schema with explicit field constraints and compatibility rules.

Scope:
- JSON schema for ticket spec and backlog spec.
- Validation engine returning structured field-level errors.
- Versioning strategy for future schema evolution.

Out of scope: CLI mutation commands themselves.

## Acceptance Criteria
- [ ] Unit tests verify schema success and failure cases for required fields type mismatches and invalid relations.
- [ ] Integration test validates real spec files in a temp repo and confirms deterministic parse and structured error output.
- [ ] Validation evidence includes schema version and representative validator output attached in ticket comments.
