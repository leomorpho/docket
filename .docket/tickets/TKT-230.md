# TKT-230: Implement transactional docket ticket apply command

## Description
Context: LLMs should create or update a fully groomed ticket in one transaction, not many mutating calls.

Objective: Add docket ticket apply command to atomically create or update title, description, ACs, labels, parent, and metadata from a spec.

Scope:
- New command: docket ticket apply --spec <file or stdin>.
- Transaction semantics for all ticket fields and AC replacement policy.
- JSON response with id, warnings, and next actions.

Out of scope: Multi-ticket backlog orchestration.

## Acceptance Criteria
- [x] Unit tests verify one single command can create or update a ticket transactionally with AC reconciliation and rollback on partial failures. (run: go test ./cmd -run TestTicketApplyCreateAndUpdateTransactional|TestTicketApplyCreateRollbackOnFailure) : Added command tests for transactional create/update, AC replacement semantics, and counter rollback when create fails.
- [x] Integration test applies valid ticket specs in temp repos and verifies markdown plus index state match expectations. (run: go test ./cmd -run TestTicketApplyIntegrationMarkdownAndIndex) : Added integration test covering spec apply -> markdown artifact + sqlite index visibility.
