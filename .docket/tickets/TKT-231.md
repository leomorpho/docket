# TKT-231: Implement transactional docket backlog apply command

## Description
Context: LLMs often need to create an epic and many children with dependencies in one operation.

Objective: Add docket backlog apply command for atomic batch creation and linking of multiple tickets from one spec.

Scope:
- New command: docket backlog apply --spec <file or stdin>.
- Supports epic, child tickets, parent assignment, labels, and dependency links.
- Reports created IDs and unresolved references with structured output.

Out of scope: Cross-repo backlog orchestration.

## Acceptance Criteria
- [x] Unit tests verify one single command can create an epic plus child backlog with stable ID mapping parent linking dependencies and invalid-reference failure behavior. (run: go test ./cmd -run TestBacklogApplyCreatesStableIDMapping|TestBacklogApplyRollbackOnPartialFailure) : Added backlog apply unit tests for stable ref->ID mapping, parent/dependency linking, and invalid-parent rollback behavior.
- [x] Integration test creates epic plus children in one apply call and asserts expected relation graph. (run: go test ./cmd -run TestBacklogApplyIntegrationRelationGraph) : Added integration test asserting created epic->children relation graph after a single backlog apply call.
- [x] Validation evidence includes backlog apply result payload and resulting ticket tree snapshot in comments. (run: DOCKET_HOME=/tmp/docket-home /home/developer/go/bin/docket show TKT-231) : Comment captures representative backlog apply JSON payload and resulting ticket tree snapshot.
