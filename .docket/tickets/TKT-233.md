# TKT-233: Harden sqlite write paths with busy timeout and bounded retry policy

## Description
Context: LLM loops trigger rapid command bursts that can expose sqlite busy failures.

Objective: Harden store writes with busy timeout and retry policy to reduce transient lock failures.

Scope:
- Configure sqlite busy timeout and safe retry behavior for write paths.
- Add internal backoff policy for known retryable lock errors.
- Ensure retries remain bounded and observable.

Out of scope: Replacing sqlite backend.

## Acceptance Criteria
- [ ] Unit tests cover retryable lock errors bounded retries and final failure propagation semantics.
- [ ] Integration test simulates concurrent mutation pressure and verifies reduced SQLITE_BUSY failures.
- [ ] Validation evidence includes before and after lock-failure rates captured in test logs and comments.
