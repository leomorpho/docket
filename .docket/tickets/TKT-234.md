# TKT-234: Add machine-readable error envelope for mutating commands

## Description
Context: LLMs need field-specific remediation guidance, not generic command failures.

Objective: Standardize machine-readable error envelope across mutating commands.

Scope:
- Add error code, field, retryable, and suggested fix fields in json output.
- Normalize validation and transition errors to shared envelope shape.
- Keep human output concise while improving json diagnostics.

Out of scope: Full i18n error catalog.

## Acceptance Criteria
- [ ] Unit tests verify stable error-envelope fields for validation transition and storage failures.
- [ ] Integration test triggers representative failures and asserts structured json diagnostics are emitted.
- [ ] Validation evidence includes failure fixtures with expected error_code and suggested_fix values.
