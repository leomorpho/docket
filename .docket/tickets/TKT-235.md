# TKT-235: Add scaffold commands for ticket and backlog spec templates

## Description
Context: LLMs perform better when given valid starter specs instead of synthesizing complex payloads from scratch.

Objective: Add scaffold commands for ticket and backlog spec templates.

Scope:
- Commands to emit valid ticket and backlog spec skeletons.
- Include concise inline guidance comments or fields.
- Ensure generated templates validate against schema out of the box.

Out of scope: Opinionated domain-specific templates.

## Acceptance Criteria
- [x] Unit tests verify scaffold output matches schema and remains parseable after regeneration. : go test ./cmd -run "TestTicketScaffoldOutputMatchesSchemaAndIsStable|TestBacklogScaffoldOutputMatchesSchemaAndIsStable" (scaffold output schema-valid and deterministic)
- [x] Integration test scaffolds then applies templates with minimal edits and confirms successful creation. : go test -v ./cmd -run TestScaffoldThenApplyTemplatesWithMinimalEdits (scaffold commands followed by minimal edits and successful ticket/backlog apply)
- [x] Validation evidence includes scaffold snapshots and apply-success logs linked in comments. : Integration test writes scaffold snapshots and apply-success logs under .docket/test-fixtures/scaffold/{ticket-scaffold.json,backlog-scaffold.json,ticket-apply-result.json,backlog-apply-result.json}; fixture paths logged in verbose output.
