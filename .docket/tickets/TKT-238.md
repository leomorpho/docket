# TKT-238: Harden ticket parser so nested markdown headings do not break Description parsing

## Description
Context: The current markdown parser treats any heading markers as top-level ticket sections. LLM-written descriptions often contain nested headings, which can accidentally truncate Description and cause validate failures.

Objective: Make ticket parsing resilient so nested markdown headings inside Description or other rich text fields do not corrupt section parsing.

Scope:
- Update parser logic to recognize only true top-level ticket sections.
- Preserve nested markdown structure in Description content.
- Add regression protection for rich markdown bodies used by LLMs.

Out of scope: Changing canonical ticket section names or rewriting historical ticket data.

Delivery notes: This is a release-blocking reliability fix for the single-command apply experience.
