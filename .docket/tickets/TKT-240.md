# TKT-240: Define screenshot proof metadata schema (`proof_title` + `note`) and durable storage

## Description
Context: CLI and UI work depend on a stable proof artifact contract.

Objective: implement canonical screenshot proof metadata and local storage APIs with strong validation.

Contract requirements: `proof_title` (required), `note` (required single narrative field), `added_at` (required RFC3339), `captured_at` (optional RFC3339), plus image metadata (relative path, mime type, size bytes, sha256, actor).

Storage: keep proof files and metadata under `.docket` runtime/ticket proof directories with deterministic IDs and path safety checks.

Likely paths: `internal/proof/*` (new), `internal/store/local/*`, shared validator/error envelope helpers.

Verify commands: `go test ./internal/... -run Proof` and `go test ./...`.

Out of scope: UI rendering and command UX (handled in dependent tickets).

Delivery notes: implement TDD-first with fixture-backed parser/storage tests.
