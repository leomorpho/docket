# TKT-241: Implement `docket proof` CLI (`add/list/remove`) with `proof_title` and `note`

## Description
Context: agents need deterministic commands to attach screenshots as ticket proofs without manual file edits.

Objective: add `docket proof add`, `docket proof list`, and `docket proof remove` commands backed by canonical proof storage.

Command contract (v1): `docket proof add <TKT-NNN> --file <path> --proof-title <text> --note <text> [--captured-at RFC3339] [--format json]`.

Behavior requirements: timestamp `added_at` recorded at write time, deterministic JSON output for automation, and stable proof IDs for subsequent list/remove operations.

Likely paths: `cmd/proof*.go` (new), proof storage layer from TKT-240, shared mutation error envelope helpers.

Verify commands: `go test ./cmd -run Proof` and `go test ./...`.

Out of scope: browser upload flows and non-image media handling.

Delivery notes: TDD-first with command snapshot fixtures.

## Acceptance Criteria
- [ ] Unit tests (written first) cover add/list/remove happy paths, required flag validation (`--proof-title`, `--note`), and deterministic JSON output.
- [ ] Integration tests attach sample image fixtures, verify persisted metadata includes `proof_title`, `note`, `added_at`, and exercise list/remove end-to-end.
- [ ] Failure paths emit machine-readable error envelopes (`error_code`, `field`, `retryable`, `suggested_fix`) for invalid proof operations.
