# TKT-241: Implement `docket proof` CLI (`add/list/remove`) with `proof_title` and `note`

## Description
Context: agents need deterministic commands to attach screenshots as ticket proofs without manual file edits.

Objective: add `docket proof add`, `docket proof list`, and `docket proof remove` commands backed by canonical proof storage.

Command contract (v1): `docket proof add <TKT-NNN> --file <path> --proof-title <text> --note <text> [--captured-at RFC3339] [--format json]`.

Likely paths: `cmd/proof*.go` (new), proof storage layer from TKT-240, shared mutation error envelope helpers.

Out of scope: browser upload flows and non-image media handling.
