# TKT-242: Expose screenshot proof metadata in `show`, `context`, and JSON ticket outputs

## Description
Context: attached proofs must be discoverable by both humans and LLMs through existing ticket inspection commands.

Objective: surface screenshot proof metadata in `docket show`, `docket context`, and JSON output contracts.

Output requirements: include proof ID, `proof_title`, `note`, timestamps, and image metadata with deterministic ordering and concise human formatting.

Likely paths: `cmd/show.go`, `cmd/context.go`, output tests, and ticket JSON output shapers.

Verify commands: `go test ./cmd -run 'Show|Context|Proof'` and `go test ./...`.

Out of scope: visual thumbnail rendering (handled by UI ticket).

Delivery notes: implement TDD-first and preserve backward compatibility when no proofs exist.

## Acceptance Criteria
- [ ] Unit tests (written first) verify proof metadata appears in show/context/json outputs with stable ordering and field names.
- [ ] Human output remains concise: proof summaries are visible but do not overwhelm existing ticket sections.
- [ ] Regression tests confirm consumers remain compatible and outputs are unchanged when a ticket has zero proofs.
