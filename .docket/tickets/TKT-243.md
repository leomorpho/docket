# TKT-243: Render screenshot proof evidence in Ducca ticket UI

## Description
Context: proof metadata is only useful if the UI visibly surfaces evidence at review time.

Objective: render screenshot proof cards/thumbnails/links in Ducca ticket detail views using canonical proof metadata.

UI requirements: display `proof_title`, `note`, timestamps, and preview/link affordances with accessible labels and robust empty/error states.

Likely paths: Ducca ticket detail components, proof view models, UI state selectors, and test fixtures.

Verify commands: project UI test suite for ticket detail views plus proof-specific component tests.

Out of scope: OCR overlays and media annotation tooling.

Delivery notes: TDD-first with visual/state regression coverage for no-proof and multi-proof cases.

## Acceptance Criteria
- [ ] UI tests (written first) verify rendering of proof cards with `proof_title`, `note`, and timestamps from canonical metadata.
- [ ] No-proof and load-error states are explicit, accessible, and do not regress existing ticket layout behavior.
- [ ] Multiple proof entries render deterministically with stable ordering and usable navigation in ticket detail views.
