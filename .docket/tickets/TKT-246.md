# TKT-246: Harden proof intake: MIME sniffing, format allowlist, and filesystem safety

## Description
Context: even in local-first mode, proof intake must reject unsafe inputs (path traversal, symlinks, unsupported formats).

Objective: enforce minimal but strong safety checks for proof ingestion.

Approach: validate regular-file input, reject symlinks and non-regular files, reject absolute/unsafe relative targets, sniff MIME from bytes, and allow only png/jpg/jpeg/webp/gif.

Optional guardrail: introduce configurable max proof size to reduce accidental repository bloat.

Likely paths: proof intake/validator modules, CLI add path, structured error envelope helpers.

TDD plan: write failing security/validation fixtures first, then implement checks.

Verify commands: `go test ./... -run 'Proof|MutationError|Security'`.
