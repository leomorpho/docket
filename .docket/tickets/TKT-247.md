# TKT-247: Define proof remove semantics and add garbage collection for unreferenced blobs

## Description
Context: deleting proof references should not accidentally remove shared binaries; binary cleanup still needs an explicit path.

Objective: make `proof remove` detach metadata references only, and add explicit `proof gc` to remove unreferenced blobs.

Approach: maintain proof reference graph, ensure remove is metadata-safe, and implement deterministic mark-and-sweep style GC for blob paths.

Likely paths: proof store/index helpers, `cmd/proof_remove*`, new `cmd/proof_gc*`, integration fixtures.

TDD plan: write failing tests for shared-blob references, remove behavior, and GC reachability before implementation.

Verify commands: `go test ./cmd -run 'Proof(Remove|GC)'` and `go test ./...`.

## Acceptance Criteria
- [ ] `docket proof remove` removes ticket proof references without deleting blobs that remain referenced elsewhere.
- [ ] `docket proof gc` removes only unreferenced blobs and reports deterministic JSON summary counts.
- [ ] Integration tests (written first) validate shared-blob remove safety and GC cleanup behavior end-to-end.
