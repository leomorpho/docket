# TKT-248: Record proof add/remove lifecycle events for integrity and auditability

Context: proof artifacts should participate in lightweight integrity tracking so add/remove actions are auditable.

Objective: emit lifecycle/ledger events for proof mutations with ticket ID, proof ID, blob hash, actor, and timestamps.

Approach: integrate proof mutation hooks with existing lifecycle/security event streams and ensure deterministic event ordering in integration harness tests.
