# TKT-249: Add proof query and filter ergonomics for automation and UI consumption

## Description
Context: proof lists need better discoverability for LLM workflows and UI integrations as counts grow.

Objective: extend proof listing with deterministic filters and sorting options.

Approach: support filters such as `--kind`, `--since`, `--actor`, `--limit`, and stable sort defaults (newest-first with deterministic tie-breaks).

Likely paths: `cmd/proof_list*`, proof index/query layer, output contract tests.

TDD plan: add failing tests for filter combinations and deterministic ordering, then implement query layer changes.

Verify commands: `go test ./cmd -run 'Proof.*List|Proof.*Filter'` and `go test ./...`.

## Acceptance Criteria
- [ ] Unit tests (written first) verify each filter flag and multi-filter combinations return expected deterministic subsets.
- [ ] JSON output includes stable ordering and pagination/limit behavior suitable for automation.
- [ ] Regression tests confirm default list output remains backward-compatible when filters are not provided.
