# TKT-250: Add lazy-loading and scalable rendering for screenshot proofs in Ducca UI

## Description
Context: ticket UI must remain responsive when many proofs or large images are attached.

Objective: implement lazy image loading and scalable proof rendering behavior in ticket detail views.

Approach: render metadata first, load previews on demand/viewport, and open full-resolution image via explicit interaction.

Likely paths: Ducca ticket detail components, proof card components, UI state selectors, visual regression fixtures.

TDD plan: add failing component/state tests for lazy-load transitions and fallback states before implementation.

Verify commands: UI unit/integration test suite plus targeted proof rendering tests.
