# TKT-257: Expose Docket hook metadata and status through MCP

## Description
Expose hook metadata and status over MCP so agents can inspect lifecycle policies without trying to run hooks directly. Keep hook execution internal to Docket and verify the contract with tests first.

## Acceptance Criteria
- [ ] The MCP server exposes hook metadata and hook status actions that report advisory versus blocking behavior and current readiness.
- [ ] Automated MCP tests verify hook metadata and status responses and confirm that normal hook execution is not exposed as a user-invoked MCP action.
