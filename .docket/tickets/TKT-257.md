# TKT-257: Expose Docket hook metadata and status through MCP

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-257` before committing. -->

## Description
Expose hook metadata and status over MCP so agents can inspect lifecycle policies without trying to run hooks directly. Keep hook execution internal to Docket and verify the contract with tests first.

## Acceptance Criteria
- [x] The MCP server exposes hook metadata and hook status actions that report advisory versus blocking behavior and current readiness. : Added MCP hook.list/hook.show/hook.status actions exposing hook event mode, blocking flag, and readiness fields.
- [x] Automated MCP tests verify hook metadata and status responses and confirm that normal hook execution is not exposed as a user-invoked MCP action. : Added MCP integration tests for hook metadata/status success paths, degraded and ready readiness states, unknown hook event errors, and preserved hook.run rejection.
