# TKT-264: Show hook readiness and active hook events in status surfaces

## Description
Surface hook state where it helps operators and agents without turning status output into noise. Prioritize concise readiness summaries and active or recently blocking hook events, and drive the design with tests first.

## Acceptance Criteria
- [ ] Status-oriented surfaces report hook readiness and active or recently blocking hook events in a concise way that does not overwhelm unrelated output.
- [ ] Automated tests cover quiet output when hooks are healthy and informative output when hooks are degraded, advisory, or actively blocking workflow progress.
