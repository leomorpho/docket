# TKT-269: Add a wrap-up command for end-of-session workflow checks

## Description
Add a single end-of-session command that gathers the highest-value completion checks before a ticket moves toward review. The command should guide the user through AC status, handoff readiness, and other workflow hygiene with tests written first.

## Acceptance Criteria
- [ ] The CLI exposes a wrap-up command that reports end-of-session readiness using ticket AC state, handoff requirements, and other completion checks.
- [ ] Automated tests cover wrap-up output for ready and not-ready tickets and verify that the command surfaces actionable next steps instead of vague failures.
