# TKT-272: Expand adapter installation with a portable-first fallback model

## Description
Refactor adapter installation so Docket always installs the shared portable surfaces first and then layers adapter-native integrations where supported. The portable baseline should center on canonical Agent Skills artifacts plus Docket CLI and MCP wiring, with adapter-specific extras added only after that path is healthy and test-covered.

## Acceptance Criteria
- [ ] Bootstrap or install follows a portable-first fallback model that always wires canonical Agent Skills, Docket CLI discovery, and MCP before any adapter-specific extras.
- [ ] Unit tests and regression tests cover installation behavior across adapters and verify clear reporting when only the portable skills, CLI, and MCP layer is available.
