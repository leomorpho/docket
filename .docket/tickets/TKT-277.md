# TKT-277: Add shell-safe groomed ticket creation flow for multi-line descriptions and ACs

## Description
Improve Docket ticket authoring so agents and humans can create a well-groomed ticket with multi-line description content and acceptance criteria without fighting shell quoting. Prefer a structured input path and explicit guidance that steers users away from brittle inline command construction, and drive the work with tests first.

## Acceptance Criteria
- [ ] Docket provides a shell-safe ticket authoring path for multi-line descriptions and multiple acceptance criteria that does not require brittle inline quoting.
- [ ] Automated tests cover successful creation of a groomed ticket with structured description and AC input, including failure cases for malformed authoring input.
