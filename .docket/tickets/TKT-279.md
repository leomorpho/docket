# TKT-279: Allow tickets in todo or in-progress to move back to backlog

## Description
Relax the workflow so tickets in todo or in-progress can be intentionally moved back to backlog when work is deferred or de-scoped. Preserve the audit trail, keep command behavior explicit, and implement the change with failing tests first so backward transitions do not regress existing workflow rules.

## Acceptance Criteria
- [ ] The configured workflow and update command allow intentional transitions from todo to backlog and from in-progress to backlog while preserving transition history and clear operator output.
- [ ] Unit tests and regression tests cover the new backward transitions and verify that unrelated workflow protections and privileged transition rules continue to behave as before.
