# TKT-283: Add a synchronized capability snapshot shared by CLI, MCP, and adapter docs

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-283` before committing. -->

## Description
Create a single synchronized capability snapshot that powers CLI discovery, MCP capability discovery, adapter-installed instruction artifacts, and Agent Skills rendering so agents receive the same guidance everywhere. The snapshot should become the source of truth for capability presentation and be introduced through failing tests first.

Likely paths:
- internal/capabilities/
- cmd/capabilities.go
- cmd/helpjson.go
- internal/mcp/handlers.go
- internal/skills/pack.go
- internal/adapters/skill_pack.go

Verify commands:
- go test ./cmd -run 'TestCapabilities|TestHelpJSON' -count=1
- go test ./internal/capabilities ./internal/skills ./internal/mcp -count=1
- go test ./cmd -run TestQuickPathIntegration -count=1

Out of scope:
- inventing new capability taxonomies beyond the canonical runtime contract
- adapter-specific packaging behavior that does not consume the shared capability snapshot
- unrelated hook or workflow runtime changes

## Acceptance Criteria
- [ ] CLI, MCP, adapter-installed discovery artifacts, and Agent Skills rendering are produced from the same synchronized capability snapshot and remain semantically aligned.
- [ ] Unit tests and regression tests detect drift between the shared capability snapshot and the CLI, MCP, Agent Skills, and adapter-specific rendered outputs.
