# TKT-284: Add native slash-command and command-alias installation where supported

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-284` before committing. -->

## Description
Add adapter-native slash-command or command-alias installation for environments that support it so Docket remains convenient for operators even when MCP is unavailable or native skill auto-discovery falls short. Treat slash commands as a convenience layer on top of canonical Agent Skills, Docket CLI, and MCP rather than as the primary portability mechanism, and drive the work with tests first.

## Acceptance Criteria
- [ ] Supported adapters install Docket-native slash commands or command aliases for key workflow and capability entry points while unsupported adapters report a clear fallback to canonical skills, Docket CLI, and MCP.
- [ ] Unit tests and regression tests cover native command installation, idempotent reinstall behavior, and adapter-specific fallback reporting when slash-command surfaces are unavailable.
