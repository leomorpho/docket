# TKT-288: Add self-describing Docket skill export surface for installer and agent discovery

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-288` before committing. -->

## Description
Add a self-describing skill export surface so Docket can emit its canonical Agent Skills package for installers, automation, or future agent-driven refresh flows. Keep the export stable, machine-readable, and driven by the same canonical skill source used everywhere else, with tests written first.

## Acceptance Criteria
- [ ] Docket exposes a stable skill export surface that emits the canonical Agent Skills package or metadata needed by installers and automation flows.
- [ ] Unit tests and regression tests verify the export output stays synchronized with the canonical skill definitions, capability snapshot, and rendered install artifacts.
