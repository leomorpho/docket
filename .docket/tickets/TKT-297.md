# TKT-297: Harden local artifact migration for crash, retry, and concurrency safety

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-297` before committing. -->

## Description
Context: File moves into .docket/local may be interrupted by crashes, concurrent runs, or multiple worktrees. Non-atomic behavior here can leave mixed/corrupt local state that is expensive to recover later.

Objective: Harden migration and runtime local artifact writes for crash safety, retryability, and concurrent execution robustness.

Approach:
1. Use atomic temp-file writes + rename where applicable.
2. Ensure migration operations are idempotent and safe to re-run.
3. Add lock/coordination strategy and tests for concurrent migration/write attempts.

Likely paths: cmd/upgrade*.go, internal/store/local/*, local artifact utility helpers, lock primitives.

Verify commands: go test ./... plus targeted concurrency/retry tests.

Out of scope: changing ticket schema semantics; network/distributed locking.

## Acceptance Criteria
- [ ] Go tests pass (run: go test ./...)
- [ ] Local artifact writes and migration steps use atomic/idempotent behavior resilient to crash/retry scenarios.
- [ ] Concurrency tests cover overlapping migration/runtime operations across parallel processes or worktrees without corrupting local state.
