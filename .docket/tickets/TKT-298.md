# TKT-298: Add doctor/check diagnostics for legacy-new artifact layout drift

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-298` before committing. -->

## Description
Context: During transition, repositories can end up in legacy-only, new-only, or mixed artifact states. Without explicit diagnostics, operators and agents lose time debugging subtle behavior.

Objective: Extend doctor/check surfaces to detect artifact layout drift and mixed-state rollout issues with precise remediation guidance.

Approach:
1. Add artifact-layout diagnostics in doctor/check for legacy/new/mixed states.
2. Include machine-readable result fields so MCP/agents can react deterministically.
3. Ensure messaging points to exact commands to repair state (upgrade/bootstrap where relevant).

Likely paths: cmd/doctor.go, cmd/check.go, cmd/*_test.go, internal diagnostics helpers.

Verify commands: go test ./... and focused doctor/check fixture tests.

Out of scope: performing migration directly from doctor/check (diagnostics only).

## Acceptance Criteria
- [ ] Go tests pass (run: go test ./...)
- [ ] Doctor/check report legacy/new/mixed artifact layout states with machine-readable fields and actionable remediation messages.
- [ ] Regression fixtures verify quiet healthy output plus explicit drift diagnostics for mismatched layout states.
