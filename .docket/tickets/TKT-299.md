# TKT-299: Add cross-version compatibility matrix for artifact layout rollout

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-299` before committing. -->

## Description
Context: Teams will run mixed Docket versions during rollout. If old binaries break on new artifact layouts (or vice versa), adoption becomes risky and rollback painful.

Objective: Guarantee forward/backward compatibility behavior between pre-migration and post-migration binaries across supported upgrade windows.

Approach:
1. Define compatibility contract for old/new binaries and artifact layout expectations.
2. Add regression harness scenarios covering old->new->old and new->old handoffs on real repo fixtures.
3. Document unsupported combinations and explicit failure modes.

Likely paths: fake-repo integration harness, upgrade tests, docs migration compatibility section.

Verify commands: go test ./... plus compatibility matrix integration tests.

Out of scope: indefinite support for all historical versions outside declared support window.

## Acceptance Criteria
- [ ] Go tests pass (run: go test ./...)
- [ ] Compatibility matrix tests cover old->new->old and new->old repo transitions within the declared support window.
- [ ] Documentation clearly defines supported version windows and deterministic failure modes for unsupported combinations.
