# TKT-300: Add canonical Docket artifact registry to prevent path/policy drift

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-300` before committing. -->

## Description
Context: Artifact paths are currently referenced from multiple code paths. Without a single source-of-truth registry, new files can bypass the tracked/local policy and drift from migration or hook enforcement.

Objective: Create one canonical artifact registry used by runtime, migration, ignore management, and hooks.

Approach:
1. Define typed registry entries for artifact path, category, tracked/local policy, and migration behavior.
2. Replace scattered hardcoded path checks with registry lookups.
3. Add tests that fail on unregistered artifact references in key subsystems.

Likely paths: internal/* path helpers, cmd/install*.go, cmd/upgrade.go, hook check commands.

Verify commands: go test ./... and registry drift tests.

Out of scope: changing artifact semantics beyond path/policy metadata.

## Acceptance Criteria
- [x] Go tests pass (run: go test ./...) : go test ./...
- [x] A single artifact registry defines path, tracked/local policy, and migration behavior for all managed Docket artifacts. : Added internal/artifacts registry with typed entries covering repo-rooted and DOCKET_HOME-managed artifacts, plus shared path and ignore helpers.
- [x] Tests fail when unmanaged/hardcoded artifact path checks bypass the registry in core lifecycle paths. : Added internal/artifacts source drift test that asserts core lifecycle files import the registry and avoid raw .docket path literals.

## Handoff
**Current state:** Implementation complete. A shared internal/artifacts registry now owns managed artifact metadata and core path helpers route through it for repo and DOCKET_HOME state.

**Decisions made:** Classified tracked vs local-only artifacts in one registry, added shared path/ignore helpers, and added a source drift test that fails if core lifecycle files bypass the registry with raw .docket literals.

**Remaining work:** Follow-up artifact-layout tickets can move remaining path callsites and add migration/local fallback behavior on top of this registry contract.
