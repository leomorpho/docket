# TKT-301: Auto-heal Docket ignore policy from init/bootstrap/install entrypoints

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-301` before committing. -->

## Description
Context: If ignore-line auto-healing only runs in upgrade, repos that skip upgrade can remain misconfigured. Init/bootstrap/install entrypoints should also reconcile ignore policy.

Objective: Ensure Docket auto-manages required local-artifact ignore lines from init/bootstrap/install and not only upgrade.

Approach:
1. Reuse one idempotent ignore reconciler across init/bootstrap/install/upgrade.
2. Ensure line-by-line insertion without duplicates and stable ordering.
3. Add regression tests for fresh and existing repos across entrypoints.

Likely paths: cmd/init.go, cmd/bootstrap.go, cmd/install*.go, shared ignore helpers.

Verify commands: go test ./... and entrypoint-specific tests.

Out of scope: migration of artifact contents.

## Acceptance Criteria
- [ ] Go tests pass (run: go test ./...)
- [ ] Init/bootstrap/install each reconcile required ignore lines idempotently without relying on upgrade execution.
- [ ] Regression tests cover fresh and existing repositories for ignore-line stability and no-duplication behavior.
