# TKT-302: Add migration rollback and recovery workflow for partial .docket/local failures

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-302` before committing. -->

## Description
Context: Migration failures are inevitable (partial writes, interrupted runs). Without explicit rollback/recovery guidance or tooling, teams can get stuck in broken mixed states.

Objective: Add deterministic rollback/recovery behavior for local artifact layout migration failures.

Approach:
1. Define recovery contract for partial migration states.
2. Add command-path or documented workflow to restore a known-good state.
3. Ensure doctor/check surfaces point to exact recovery commands.

Likely paths: cmd/upgrade.go, cmd/doctor.go, docs migration section, migration helpers.

Verify commands: go test ./... plus failure-injection recovery tests.

Out of scope: long-term support beyond declared compatibility window.

## Acceptance Criteria
- [ ] Go tests pass (run: go test ./...)
- [ ] Recovery workflow restores deterministic known-good state from partial migration failures.
- [ ] Doctor/check remediation output points directly to rollback/recovery commands and tested outcomes.
