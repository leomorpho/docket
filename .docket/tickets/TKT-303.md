# TKT-303: Add CI artifact policy drift checks for local-vs-tracked invariants

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-303` before committing. -->

## Description
Context: As the codebase evolves, new local artifacts may be introduced outside .docket/local unless CI enforces policy centrally.

Objective: Add CI-facing drift checks that fail when local artifacts escape canonical local paths or tracked-required expectations drift.

Approach:
1. Add a policy assertion test that validates artifact registry invariants.
2. Fail on known local artifacts outside .docket/local and on missing required tracked artifacts in enforcement sets.
3. Integrate into default test paths so drift is caught early.

Likely paths: cmd/*_test.go or internal policy tests, artifact registry package.

Verify commands: go test ./... and targeted drift check tests.

Out of scope: runtime migration behavior changes.

## Acceptance Criteria
- [ ] Go tests pass (run: go test ./...)
- [ ] CI-facing tests enforce local-vs-tracked artifact invariants and fail when local artifacts escape canonical namespaces.
- [ ] Policy drift checks are integrated into standard test execution so regressions fail fast.
