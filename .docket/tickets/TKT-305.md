# TKT-305: Implement dedicated .gitignore reconciliation unit for Docket local artifacts

## Description
Context: Ignore reconciliation deserves independent implementation/tests and clearer ownership than a bundled migration ticket.

Objective: Implement idempotent line-by-line .gitignore reconciliation for local artifacts as a dedicated unit.

Approach:
1. Build ignore reconciler with stable ordering and no duplicate lines.
2. Cover legacy+canonical transition lines during rollout window.
3. Expose reusable helper for init/bootstrap/install/upgrade integration.

Likely paths: shared cmd/install/upgrade helpers and tests.

Verify commands: go test ./... and ignore-specific fixture tests.

Out of scope: moving artifact files.
