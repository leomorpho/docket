# TKT-306: Split pre-commit tracked-required artifact enforcement into dedicated policy unit

## Description
Context: Pre-commit enforcement currently bundles multiple policies; separate tracked-required checks reduce ambiguity and improve remediation UX.

Objective: Implement tracked-required artifact enforcement as a dedicated pre-commit policy unit.

Approach:
1. Validate required tracked artifacts are present and staged when expected.
2. Emit deterministic remediation guidance for missing/untracked files.
3. Add focused tests for pass/fail cases.

Likely paths: pre-commit hook checker commands and tests.

Verify commands: go test ./... and hook-focused tests.

Out of scope: blocking staged local-only artifacts.

## Acceptance Criteria
- [ ] Go tests pass (run: go test ./...)
- [ ] Pre-commit tracked-required policy fails deterministically when required tracked artifacts are missing or unstaged.
- [ ] Remediation guidance includes explicit commands to add/stage required tracked artifacts.
