# TKT-307: Split pre-commit local-only artifact rejection into dedicated policy unit

## Description
Context: Blocking local-only artifacts in commits has distinct detection/remediation logic and should be independently testable.

Objective: Implement local-only artifact staging rejection as a dedicated pre-commit policy unit.

Approach:
1. Detect staged local-only artifacts from canonical policy set.
2. Provide deterministic remediation guidance (`git rm --cached`, ignore reconciliation).
3. Add focused pass/fail and message-stability tests.

Likely paths: pre-commit hook checker commands and tests.

Verify commands: go test ./... and hook-focused tests.

Out of scope: required tracked artifact enforcement.

## Acceptance Criteria
- [ ] Go tests pass (run: go test ./...)
- [ ] Pre-commit local-only policy rejects staged local artifacts with deterministic detection across canonical/transition paths.
- [ ] Remediation guidance includes explicit commands for unstage/remove-cached plus ignore reconciliation.
