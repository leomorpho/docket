# TKT-311: [Epic] Secure approval and activation of editable workflow policy

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-311` before committing. -->

## Description
Keep workflow definitions human-editable in the repo while moving approval and activation authority into the secure area. The result should let operators customize workflow behavior without letting agents silently weaken the active policy mid-run.

## Acceptance Criteria
- [ ] All child tickets under this epic land in dependency order with deterministic tests and secure-area approval metadata for the active workflow definition.
- [ ] Editable repo workflow config and secure approval metadata are clearly separated so humans can change policy while enforcement still pins an approved definition.
- [ ] Runtime enforcement surfaces explain which workflow definition is active, approved, pending, or mismatched without relying on hardcoded workflow state names.
