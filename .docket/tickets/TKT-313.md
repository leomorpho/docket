# TKT-313: Define workflow config v1 schema for semantic roles, per-state behavior, and presentation metadata

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-313` before committing. -->

## Description
Add a versioned workflow config schema that separates semantic workflow behavior from presentation metadata. The schema should support state roles, per-state flags such as open or terminal or startable or reviewable or blocks_dependents, transition rules, and presentation fields like label or column ordering. Implement it from failing tests first so custom workflows can be validated deterministically.

## Acceptance Criteria
- [ ] Tests are added or updated first to validate workflow config v1 schema success cases and field-level failure cases for roles, per-state semantics, transitions, and presentation metadata.
- [ ] Docket can load and validate a versioned workflow config that separates semantic workflow behavior from presentation metadata without relying on hardcoded state-name assumptions.
- [ ] Go tests pass for the schema and validation changes.
