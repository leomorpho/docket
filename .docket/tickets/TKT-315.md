# TKT-315: Implement a shared workflow evaluator that resolves transitions, roles, startability, and dependency blocking from config

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-315` before committing. -->

## Description
Build one workflow evaluator that consumes workflow config and answers the semantic questions Docket needs: allowed transitions, state roles, startability, reviewability, and whether a ticket in a given state still blocks dependents. Drive it from failing tests first and make the API broad enough to support future approval or AC-based blocking without another redesign.

## Acceptance Criteria
- [ ] Tests are added or updated first to cover transition evaluation, role lookup, per-state flags, startability, reviewability, and blocker-state semantics across default and custom workflow shapes.
- [ ] Docket exposes a shared workflow evaluator that derives semantic behavior from config rather than from hardcoded state names or naive blocked_by checks.
- [ ] Go tests pass for the workflow evaluator and its fixtures.
