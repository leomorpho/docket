# TKT-319: Refactor docket start to use config-defined startable states and evaluator-based readiness

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-319` before committing. -->

## Description
Remove the hardcoded backlog and todo intake assumption from docket start and have selection come from config-defined startable semantics plus evaluator-based readiness. Implement from failing tests first so repos with renamed intake states can still use start without code changes.

## Acceptance Criteria
- [ ] Tests are added or updated first to prove docket start works for the default workflow and for a custom workflow whose startable intake states are not named backlog or todo.
- [ ] Docket start selects candidates from config-defined startable semantics and evaluator-based readiness rather than hardcoded intake state names.
- [ ] Go tests pass for docket start selection changes.
