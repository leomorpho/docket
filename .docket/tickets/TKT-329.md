# TKT-329: Refactor ticket-apply and backlog-apply validation to accept config-defined workflow states

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-329` before committing. -->

## Description
The apply schemas and validators currently hardcode a fixed workflow state enum, which will conflict with configurable workflow definitions. Write the failing tests first, then update apply-schema validation so ticket and backlog apply flows validate states against the configured workflow definition rather than a baked-in list of names.

## Acceptance Criteria
- [ ] Tests are added or updated first to prove ticket-apply and backlog-apply accept default and custom workflow state names from configuration and reject unknown states deterministically.
- [ ] Apply-schema validation and runtime ticket/backlog apply flows validate state names against workflow config instead of a hardcoded fixed enum.
- [ ] Go tests pass for the apply-schema and apply-command changes.
