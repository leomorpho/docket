# TKT-330: Write and refresh the default workflow config artifact from init, bootstrap, and install entrypoints

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-330` before committing. -->

## Description
Make default workflow config materialization explicit so repositories get a concrete editable workflow artifact rather than implicit behavior. Implement the work from failing tests first and ensure init, bootstrap, and install can create or refresh the default workflow config deterministically without clobbering explicit human customizations.

## Acceptance Criteria
- [ ] Tests are added or updated first to cover fresh repositories, repositories missing workflow config, and repositories with existing explicit workflow config across init, bootstrap, and install entrypoints.
- [ ] Init, bootstrap, and install write or refresh the shipped default workflow config artifact deterministically without overwriting explicit human-customized workflow definitions.
- [ ] Go tests pass for the workflow-config entrypoint changes.
