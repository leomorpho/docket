# TKT-331: Add an explicit operator command flow to approve and activate workflow policy changes

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-331` before committing. -->

## Description
The secure-area model needs a concrete operator action for reviewing and activating an editable workflow definition. Write the failing tests first, then implement an explicit approval and activation command flow that records the approved workflow identity in the secure area, reports what changed, and behaves predictably across advisory and enforced rollout modes.

## Acceptance Criteria
- [ ] Tests are added or updated first to cover approving a workflow definition, re-approving an unchanged definition, and rejecting or surfacing mismatched activation inputs.
- [ ] Docket exposes an explicit operator command flow to approve and activate workflow policy changes using secure-area approval metadata rather than implicit side effects.
- [ ] Go tests pass for the approval and activation command flow across advisory and enforced modes.
