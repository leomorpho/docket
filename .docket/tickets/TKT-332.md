# TKT-332: Add doctor and status diagnostics for workflow config validity, resolved source, and mismatch reporting

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-332` before committing. -->

## Description
Operators still lack first-class visibility into which workflow source Docket resolved at runtime, especially in repositories like this one that currently rely on the legacy `states` shape and compatibility fallback instead of an explicit `workflow` stanza. Keep this ticket focused on deterministic doctor/status reporting for config validity, resolved source, fallback mode, and any mismatch between surfaces.

## Acceptance Criteria
- [ ] Tests are added or updated first to cover healthy, invalid, states-only compatibility fallback, and workflow-source mismatch scenarios in doctor and status surfaces.
- [ ] Doctor and status surfaces report workflow config validity, resolved workflow source, compatibility fallback mode, and mismatch details deterministically for both human and machine consumers.
- [ ] Go tests pass for the workflow diagnostics changes.
