# TKT-332: Add doctor and status diagnostics for workflow config validity, resolved source, and mismatch reporting

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-332` before committing. -->

## Description
Operators need first-class visibility into whether the workflow config is valid, which workflow definition Docket resolved at runtime, and whether runtime surfaces are falling back or disagreeing about that source. Implement the diagnostics from failing tests first and make the output deterministic for humans and machine consumers.

## Acceptance Criteria
- [ ] Tests are added or updated first to cover healthy, invalid, missing-config fallback, and workflow-source mismatch scenarios in doctor and status surfaces.
- [ ] Doctor and status surfaces report workflow config validity, resolved workflow source, and fallback or mismatch details deterministically for both human and machine consumers.
- [ ] Go tests pass for the workflow diagnostics changes.
