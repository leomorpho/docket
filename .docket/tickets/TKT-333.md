# TKT-333: Invalidate caches and indexed readiness when workflow config changes

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-333` before committing. -->

## Description
Config-defined workflow semantics will drift if indexes or cached readiness decisions survive workflow edits unchanged. Write the failing tests first, then ensure workflow-config changes invalidate the relevant caches and indexed readiness so downstream commands re-evaluate against the active config.

## Acceptance Criteria
- [ ] Tests are added or updated first to prove workflow-config edits invalidate cached or indexed readiness so subsequent commands observe the new policy immediately.
- [ ] Docket invalidates or refreshes workflow-dependent caches and indexed readiness when workflow config changes.
- [ ] Go tests pass for the workflow-config invalidation behavior.
