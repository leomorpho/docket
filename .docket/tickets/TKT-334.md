# TKT-334: Add a compatibility matrix for old repos, default workflow config, and custom workflow config

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-334` before committing. -->

## Description
The workflow-config rollout needs regression coverage across repositories with no explicit workflow config, with the shipped default config, and with custom workflow definitions. Implement the matrix from failing tests first so workflow compatibility is locked down without inventing a heavy migration program.

## Acceptance Criteria
- [ ] Tests are added first to cover repositories without explicit workflow config, repositories using the shipped default workflow artifact, and repositories with custom workflow definitions.
- [ ] The compatibility matrix is reusable in standard test execution so workflow rollout regressions are caught without manual setup.
- [ ] Go tests pass for the workflow compatibility matrix coverage.
