# TKT-335: Update start, help, and user guidance surfaces to describe workflow roles instead of hardcoded state names

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-335` before committing. -->

## Description
Even after the engine is config-driven, help and guidance text can still leak assumptions like backlog, todo, in-progress, or in-review. Write the failing tests first, then update start and other guidance surfaces so they explain semantic workflow roles and config-driven readiness rather than literal state names where those names are no longer guaranteed.

## Acceptance Criteria
- [ ] Tests are added or updated first to cover guidance output under default and custom workflow shapes without depending on hardcoded state-name strings.
- [ ] Start, help, and related user guidance surfaces describe workflow roles and config-driven readiness semantics instead of relying on fixed state names where customization is allowed.
- [ ] Go tests pass for the workflow guidance updates.
