# TKT-337: [Epic] Experimental phase 2 local secure-storage improvements

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-337` before committing. -->

## Description
Treat local secure-storage work as an experimental phase 2 hardening track, not as part of the near-term product-critical roadmap. This epic may explore wrapped-key keystores, OS-backed protection, humane reset flows, and related DX improvements after the simpler warning-only versus enforced rollout has proven useful. None of this work should block workflow config, warning-only defaults, or the simple repo-level enforcement toggle.

## Acceptance Criteria
- [ ] Any child tickets that are implemented stay explicitly experimental and do not block the core warning-only versus enforced rollout.
- [ ] Docket can improve local secure-storage behavior without collapsing per-human identity boundaries or forcing keystore complexity into the core product path.
- [ ] Unlock, reset, recovery, and storage docs remain clearly optional follow-on behavior while secure enforcement itself stays intentionally small.
