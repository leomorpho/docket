# TKT-339: Refactor file keystore to use a generated data key with password-backed protector fallback

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-339` before committing. -->

## Description
Optional phase 2 refactor for moving the file keystore to a generated data-key model with password-backed wrapping as a fallback provider. This is intentionally deferred until the simpler experimental security rollout is proven useful.

## Acceptance Criteria
- [ ] Failing tests are added first for generated data-key handling, password-backed wrapping, and unlock behavior under the new protector model.
- [ ] The file keystore encrypts payloads with a generated data key while preserving password-based unlock as a fallback protector.
- [ ] Go tests pass for the password-backed protector refactor.
