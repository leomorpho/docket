# TKT-341: Implement macOS Keychain protector for wrapped keystore data keys

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-341` before committing. -->

## Description
Optional phase 2 macOS-only enhancement for using Keychain-backed protection. This is explicitly later hardening work, not part of the near-term experimental security rollout.

## Acceptance Criteria
- [ ] Failing tests are added first for macOS protector provisioning, unwrap behavior, and unsupported-environment handling.
- [ ] On macOS, Docket can use Keychain-backed protection to wrap and unwrap the local keystore data key without relying on a Docket password.
- [ ] Go tests pass for the macOS protector implementation, with non-macOS paths gated cleanly.
