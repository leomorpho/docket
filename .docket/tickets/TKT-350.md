# TKT-350: Add simple opt-in security enforcement switch with warning-only-by-default behavior

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-350` before committing. -->

## Description
Introduce a simple repo-shared configuration-backed enforcement switch so security-sensitive behavior is warning-only by default and only becomes enforced when a team explicitly opts in. This rollout starts only after the config-backed workflow evaluator foundation is in place so enforcement behavior does not depend on stale hardcoded workflow rules. Keep the model minimal and avoid a larger mode matrix unless real usage proves it is needed.

## Acceptance Criteria
- [ ] Failing tests are added first for warning-only-by-default behavior and explicitly enabled enforcement behavior across the security-enforcement configuration surface.
- [ ] Docket supports a simple repo-shared opt-in enforcement switch instead of assuming secure blocking is always active.
- [ ] Go tests pass for the simplified enforcement configuration behavior.
