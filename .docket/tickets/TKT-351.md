# TKT-351: Default experimental security enforcement to warning-only behavior until explicitly enabled

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-351` before committing. -->

## Description
Change the default rollout so experimental secure enforcement is not active unless a repo explicitly enables it. When enforcement is not enabled, privileged and workflow-policy checks should warn clearly instead of hard-blocking normal mutation flows.
