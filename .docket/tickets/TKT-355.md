# TKT-355: Add structured warning signals to JSON and MCP mutation surfaces when enforcement is not enabled

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-355` before committing. -->

## Description
Humans can read warning text, but agents need structured signals. Add machine-readable warning behavior to JSON and MCP mutation surfaces so commands that would block under enabled enforcement return deterministic warnings instead of opaque success or failure states.

## Acceptance Criteria
- [ ] Failing tests are added first for JSON and MCP mutation flows that proceed with structured warnings when enforcement is not enabled.
- [ ] Machine-facing surfaces return deterministic warnings when privileged or policy-sensitive actions are allowed because enforcement is still warning-only.
- [ ] Go tests pass for warning-mode JSON and MCP signaling.
