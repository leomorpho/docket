# TKT-357: Set brand-new repos to ask about experimental secure mode and fall back to warning-only when declined

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-357` before committing. -->

## Description
Define onboarding defaults for new repos so Docket asks whether to enable the experimental secure mode during setup. If the user declines, Docket should simply stay in warning-only behavior rather than silently enabling enforcement or leaving the mode ambiguous.

## Acceptance Criteria
- [ ] Failing tests are added first for init, bootstrap, or install flows prompting for experimental secure mode and defaulting to warning-only behavior when declined.
- [ ] Brand-new repos ask whether to enable experimental secure enforcement and otherwise remain warning-only.
- [ ] Go tests pass for new-repo secure-mode defaults.
