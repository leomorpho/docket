# TKT-358: Add backward-compatible upgrade behavior for existing repos moving to warning-only-by-default security enforcement

## Description
Existing repos will pick up a newer Docket binary while still carrying older repo files and assumptions. Add explicit compatibility handling so older repos land in a coherent warning-only-by-default state without broken secure flows or stale hard-block expectations.

## Acceptance Criteria
- [ ] Failing tests are added first for old repos upgrading into the warning-only-by-default rollout with existing workflow, secure, and install artifacts present.
- [ ] Docket upgrades older repos into a coherent warning-only-by-default security model without requiring manual repair of existing files before normal use.
- [ ] Go tests pass for backward-compatible upgrade behavior.
