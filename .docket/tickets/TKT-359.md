# TKT-359: Record and surface warning-mode privileged events in visible human and LLM-readable history surfaces

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-359` before committing. -->

## Description
Warning-mode telemetry is only useful if humans and agents can actually see it. Record warning-mode privileged and policy-sensitive events distinctly and surface them through visible CLI, MCP, or status history rather than burying them in unread local logs. This is explicitly later visibility work and should not block the simpler warning semantics, status surfaces, or repo toggle flows.

## Acceptance Criteria
- [ ] Failing tests are added first for recording warning-mode privileged events and surfacing them in readable CLI and machine-facing history outputs.
- [ ] Warning-mode privileged and policy-sensitive events are recorded distinctly and surfaced in human-readable and machine-readable history views.
- [ ] Go tests pass for visible warning-event recording and surfacing.
