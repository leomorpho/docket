# TKT-365: Add simple `docket secure enable` and `docket secure disable` commands for existing repos

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-365` before committing. -->

## Description
Give existing repositories a first-class but very small human command flow to enable or disable experimental secure enforcement without hand-editing repo files. Keep the interaction to a simple repo-level on/off switch and avoid introducing policy wizards or multi-step setup.

## Acceptance Criteria
- [ ] Failing tests are added first for `docket secure enable` and `docket secure disable` on an existing repo.
- [ ] Existing repos can turn experimental secure enforcement on or off through explicit Docket commands instead of manual config edits.
- [ ] Go tests pass for the simple existing-repo enforcement toggle flow.
## Likely Paths
- `cmd/secure.go`
- `cmd/init.go`
- `cmd/status.go`
- `docs/security-model.md`
- `.docket/config.json`
## Verify Commands
- `go test ./cmd/...`
- `go test ./...`
## Out of Scope
- Prompting brand-new repos during setup; that belongs in `TKT-357`.
- Defining the enforcement-setting storage model; that belongs in `TKT-361`.
- Building a multi-step policy-management or approval wizard.
