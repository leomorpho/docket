# TKT-366: Allow web and TUI ticket completion actions to move tickets to done under warning-only or enforced behavior

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-366` before committing. -->

## Description
Implement the actual web and TUI mutation path for moving tickets to done so agents and humans can continue work through the UI. In warning-only mode the action should proceed with clear warning semantics; in enforced mode it should block or require the enforced path consistently.

## Acceptance Criteria
- [ ] Failing tests are added first for web and TUI completion actions that move tickets to done under warning-only and enforced behavior.
- [ ] Web and TUI completion actions can move tickets to done when the repo is in warning-only mode and apply the enforced rule consistently when enforcement is enabled.
- [ ] Go tests pass for UI and TUI done-transition behavior, including deterministic warning or blocking semantics.
## Likely Paths
- `web/src/`
- `internal/tui/`
- `internal/mcp/`
- `cmd/update.go`
- `cmd/status.go`
## Verify Commands
- `go test ./internal/tui/...`
- `go test ./cmd/...`
- `go test ./...`
## Out of Scope
- CLI-only `update --state done` behavior; that belongs in `TKT-251`.
- Web and TUI status visibility work that does not change the mutation path; that belongs in `TKT-356`.
