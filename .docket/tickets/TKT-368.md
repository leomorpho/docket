# TKT-368: Refactor update command lifecycle side effects to use config-defined active review and completed roles

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-368` before committing. -->

## Description
The `update` command still branches on literal states such as `in-progress`, `in-review`, `done`, and `archived` when deciding lifecycle side effects and managed-run checks. Write failing tests first, then drive those behaviors from the configured graph plus fixed workflow roles so custom workflows can rename states or add intermediary active and review states without silently losing start, finish, or completion behavior.

Likely paths:
- cmd/update.go
- cmd/update_test.go
- internal/workflow/service.go
- internal/ticket/config.go

Verify commands:
- go test ./cmd -run 'TestUpdateCmd.*(CustomWorkflow|ManagedRun|Done|Review)'
- go test ./cmd ./internal/workflow

Out of scope:
- Redesigning human-only completion policy.
- UI or MCP-specific lifecycle messaging unless required by the command path.

## Acceptance Criteria
- [x] Tests are added first to cover update lifecycle behavior under at least one custom workflow that uses non-default active review and completed state names and one flow with multiple states sharing a semantic role. : Coverage present in cmd/update_test.go for custom workflow and managed-run role paths; targeted suite TestUpdateCmd.*(CustomWorkflow|ManagedRun|Done|Review) passes.
- [x] Update command lifecycle side effects and managed-run checks use graph and workflow-role helpers instead of literal state-name branching where semantics should be configurable. : cmd/update.go resolves lifecycle checks via cfg.StateHasRole and workflow-derived targets; no literal in-progress/in-review/done branching for those semantics.
- [x] Go tests pass for the affected update command paths. : Ran: go test ./cmd -run 'TestUpdateCmd.*(CustomWorkflow|ManagedRun|Done|Review)' -count=1 (pass), and go test ./internal/workflow -run 'Test.*(Start|Finish|CustomWorkflow)' -count=1 (pass).
