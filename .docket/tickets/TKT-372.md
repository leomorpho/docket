# TKT-372: Keep scaffold JSON output pure when global human skill hints are enabled

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-372` before committing. -->

## Description
The new global human skill hint is leaking into scaffold command output that tests expect to be pure JSON, breaking `TestScaffoldThenApplyTemplatesWithMinimalEdits` and `TestBacklogScaffoldOutputMatchesSchemaAndIsStable`. Write failing tests first, then ensure machine-readable scaffold output stays clean while preserving the human-only skill hint on appropriate command surfaces.

Likely paths:
- cmd/root.go
- cmd/discovery_hint.go
- cmd/scaffold_test.go
- cmd/scaffold_integration_test.go
- ticket/backlog scaffold command wiring

Verify commands:
- go test ./cmd -run 'Test(ScaffoldThenApplyTemplatesWithMinimalEdits|BacklogScaffoldOutputMatchesSchemaAndIsStable|TicketScaffoldOutputMatchesSchemaAndIsStable)'
- go test ./cmd

Out of scope:
- Removing the global skill hint entirely.
- Changing JSON payload structure for scaffold commands beyond keeping it parseable.

## Acceptance Criteria
- [ ] Tests are added first to reproduce the scaffold JSON parsing failures caused by appended human skill hints.
- [ ] Ticket and backlog scaffold commands emit pure machine-readable JSON in JSON-oriented flows while preserving the human-only skill hint on appropriate human output surfaces.
- [ ] Go tests pass for the scaffold command regression coverage.
