# TKT-373: Clean up workflow docs and generated templates for role-based semantics

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-373` before committing. -->

## Description
Update the remaining user-facing docs and generated template content that still describe the default workflow state names as if they were universal product rules. Rewrite these surfaces so they clearly distinguish default examples from config-defined workflow behavior, and make agent-facing guidance refer to the repo's configured active, review, and completed states rather than literal names.

## Acceptance Criteria
- [ ] README, CLAUDE, templates/CLAUDE, and the main workflow/security docs no longer present default state names as universal product law.
- [ ] Generated or installed agent guidance uses config-aware wording for active, review, and completed states wherever possible.
- [ ] Tests or fixtures that assert generated doc/template content are updated and pass.
