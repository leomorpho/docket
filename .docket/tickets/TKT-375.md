# TKT-375: Add one optional reviewer pass with a single capped fix-review loop for completed ticket runs

## Description

This ticket should extend the orchestrator with one optional review phase while preserving the existing module boundaries: reviewer invocation behind an adapter seam, required changes returned as structured data, and final authority still owned by the validator and orchestrator layers.
