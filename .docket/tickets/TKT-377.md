# TKT-377: Gate successful agent runs on final validation, clean repo state, and ticket transitions

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-377` before committing. -->

## Description
Implement the authoritative closeout logic for agent-managed runs. When a Codex session reports success, Docket must verify the result, validate the ticket, check the commit and repo state, and only then advance the ticket.

This ticket should own final authority checks only. It should consume structured run results and repo state, then decide whether the ticket can advance. It should not know how Codex was launched or how terminal output was parsed.

Likely paths:
- `cmd/`
- `internal/agentrun/`
- `internal/workflow/`
- `internal/store/`
- `internal/git/`
- `internal/security/`

Verify commands:
- `go test ./cmd ./internal/...`
- `docket validate TKT-377`

Out of scope:
- Ticket selection or queueing.
- Reviewer-specific orchestration.
- Parsing free-form transcript prose as authority.
- Codex adapter launch logic.

This ticket should keep the post-run rules strict and simple so later queueing logic can rely on one authoritative success path.

## Acceptance Criteria
- [x] Tests verify that a reported done result is only accepted when the expected commit exists required validation passes and repo state is acceptable. : Added tests in internal/agentrun/validate/service_test.go for mismatched RESULT ticket IDs and commits outside managed branch reachability.
