# TKT-378: Define the run contract, lifecycle states, and machine-readable result format for agent ticket runs

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-378` before committing. -->

## Description
Design and implement the small orchestration contract that every Docket-managed agent run will follow. Keep this intentionally narrow: one ticket, one fresh session, one final RESULT line, and a small set of lifecycle states that Docket can reason about deterministically.

Include the state model, result-line grammar, any persisted run metadata shape, and tests that lock the contract before process-launch work begins. This ticket should define shared types and result schemas only; it should not pull in process-launch, git, or orchestration behavior.

## Acceptance Criteria
