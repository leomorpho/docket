# TKT-380: Implement a one-ticket Codex runner that creates a fresh worktree and launches a fresh session

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-380` before committing. -->

## Description
Build the first executable Codex implementation adapter for one ticket only. Given a ticket ID, Docket should create or bind the dedicated worktree, launch a fresh Codex session in yolo mode with the run contract from the contract ticket, and persist the basic run record.

This ticket should isolate Codex-specific launch details and per-run environment preparation behind a clear adapter boundary. It should not absorb monitoring, validation, queueing, or reviewer logic.

Likely paths:
- cmd/
- internal/agentrun/
- internal/git/
- internal/security/

Verify commands:
- go test ./cmd ./internal/...
- docket validate TKT-380

Out of scope:
- Queueing multiple tickets.
- Reviewer or planner loops.
- Persisting long-lived agent memory across runs.
- Parsing terminal results beyond the launch-time contract handoff.
